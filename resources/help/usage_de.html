TSCRT Benutzerhandbuch

TSCRT ist ein Terminal-Emulator für SSH2 und serielle Konsolen. Dieses Handbuch führt durch alle wichtigen Funktionen. Zu einem Abschnitt springen:

Sitzungen

Eine Sitzung ist eine Verbindungsdefinition — entweder SSH oder seriell.

  • Sitzungen → Sitzungen verwalten… — Sitzungen hinzufügen, bearbeiten, duplizieren oder löschen. Sitzungen sind in einem Baum im linken Dock gruppiert.
  • SSH erfordert Host, Port, Benutzername und entweder ein Passwort oder eine Privatschlüsseldatei.
  • Seriell erfordert einen Gerätenamen (z. B. COM3 unter Windows, /dev/cu.usbserial-xxx unter macOS), Baudrate, Daten-/Stopp-/Paritätsbits und Flusskontrolle.
  • Doppelklicken Sie auf eine Sitzung im linken Dock, um sie in einem neuen Tab zu öffnen.

Sitzungseditor (drei Tabs)

Der Sitzungseditor ist in drei Tabs gegliedert:

  • Verbindung — Host/Port/Benutzername/Passwort/Schlüssel (SSH) oder Gerät/Baud/Leitungseinstellungen (seriell). Dieser Tab wird zuerst geöffnet, damit die am häufigsten geänderten Felder sofort sichtbar sind.
  • Allgemein — Sitzungsname, Verbindungstyp, sitzungsspezifischer Terminaltyp (z. B. xterm-256color, vt100), Umschalter für Sitzungsprotokollierung und Vollbild-Anzeigeoptionen. Wenn der Terminaltyp leer bleibt, wird die globale Voreinstellung aus Einstellungen → Präferenzen verwendet.
  • Erweitert — Einstellungen zur automatischen Wiederverbindung (maximale Versuche, Basisverzögerung) und SSH-Keepalive (Intervall, TCP-Keepalive). Die Keepalive-Gruppe ist bei seriellen Sitzungen ausgeblendet.

Tabs & Mehrfenster

  • Jede geöffnete Verbindung lebt in einem eigenen Tab. Der Tab-Titel zeigt den Sitzungsnamen; ein kleines × neben dem Namen schließt nur diesen Tab.
  • Ctrl+Alt+← / → — zum vorherigen / nächsten Tab wechseln.
  • F11 — Vollbild umschalten (blendet Dock und Menüleiste aus). Alle Tastenkürzel funktionieren weiterhin im Vollbildmodus.

Tab-Kontextmenü (Rechtsklick)

Rechtsklick auf einen Tab-Titel, um Folgendes aufzurufen:

  • Umbenennen — den angezeigten Tab-Titel ändern.
  • Duplizieren — dieselbe Sitzung in einem neuen Tab öffnen.
  • Snapshot ausführen → — einen Snapshot auf der Sitzung dieses Tabs ausführen.
  • Anheften / Lösen — angeheftete Tabs werden beim nächsten Start automatisch wiederhergestellt.
  • In neuem Fenster ablegen — den Tab in ein brandneues TSCRT-Fenster verschieben. Die SSH-/serielle Sitzung bleibt verbunden — kein Trennen/Wiederverbinden. Deaktiviert, wenn nur ein Tab vorhanden ist.
  • Tab mit Peer-Tab verknüpfen… — diesen Tab mit einem Tab in einer anderen TSCRT-Instanz auf derselben Maschine koppeln. Das Aktivieren eines der beiden Tabs wechselt auch den Peer-Tab im anderen Fenster. Nützlich für Side-by-Side-Layouts (z. B. Konsole auf dem linken Monitor, Log-Viewer auf dem rechten).
  • Tab-Verknüpfung aufheben — erscheint anstelle von Verknüpfen, wenn der Tab bereits einen Peer hat; entfernt die Kopplung.
  • Schließen — den Tab schließen.

Tabs zwischen Fenstern ziehen

Ziehen Sie einen Tab aus der Tab-Leiste und legen Sie ihn auf einem anderen TSCRT-Fenster ab, um die Sitzung zu übertragen. Wenn Sie außerhalb eines TSCRT-Fensters ablegen, wird automatisch ein neues Fenster erstellt. Die aktive Verbindung bleibt dabei erhalten.

Tab-Verknüpfungen zwischen Instanzen

Tab-Verknüpfungen werden im Profil gespeichert und automatisch wieder hergestellt, wenn beide TSCRT-Instanzen das nächste Mal gleichzeitig laufen. Verknüpfungen überstehen Neustarts. Die Peer-Erkennung erfolgt über einen rein lokalen IPC-Kanal; es entsteht kein Netzwerkverkehr.

Geteilte Bereiche / Eingabe-Broadcast / Auto-Wiederverbindung

Geteilte Bereiche

Ein einzelner Tab kann mehrere Terminalbereiche enthalten, die in einem rekursiven Teilerbaum angeordnet sind. Das Teilen verwendet das Profil des aktiven Bereichs wieder, sodass der neue Bereich auf demselben Host landet.

  • Ctrl+Shift+H — horizontal teilen (nebeneinander).
  • Ctrl+Shift+V — vertikal teilen (oben / unten).
  • Ctrl+Shift+W — den aktiven Bereich schließen. Das Schließen des letzten Bereichs schließt den Tab.
  • Teilungen sind rekursiv: Eine horizontale Teilung gefolgt von einer vertikalen Teilung erzeugt ein Gitter.
  • Der aktive Bereich ist mit einem 1 px breiten blauen Rahmen markiert. Mausklicks und Tastaturfokus aktualisieren ihn automatisch.
  • Ziehen Sie den Teiler-Griff, um die Größe zu ändern.
  • Suchleiste, Markierung, Schleife und die Befehlszeile wirken immer auf den aktiven Bereich.

Eingabe-Broadcast

Tastenanschläge über mehrere Bereiche spiegeln — nützlich für die Verwaltung einer Serverflotte.

  • Umschalten mit Ansicht → Eingabe senden oder Ctrl+Shift+B.
  • Während aktiviert erhält jeder Bereich einen orangefarbenen Rahmen.
  • Tastenanschläge, Einfügen, IME-Eingabe und Mausberichte von einem Bereich werden an jeden anderen Bereich weitergeleitet.
  • Der Schalter gilt pro Tab.

Auto-Wiederverbindung und SSH-Keepalive

Im Tab Erweitert des Sitzungseditors konfiguriert.

  • Automatisch wiederverbinden — die Sitzung nach einem Abbruch wiederherstellen. Manuell geschlossene Tabs werden nicht wiederverbunden.
  • Maximale Versuche — Abbruchgrenze. 0 = unbegrenzt (hartes Limit bei 100).
  • Basisverzögerung (ms) — anfänglicher Backoff (mindestens 250 ms). Jeder erneute Versuch verdoppelt sich, Obergrenze 30 s, mit 0–250 ms Jitter.
    Beispiel mit Basis 500: 500 ms → 1 s → 2 s → 4 s → 8 s → 16 s → 30 s …
  • SSH-Keepalive (s) (nur SSH) — libssh2-Keepalive-Intervall. Empfohlen 30 s.
  • TCP-Keepalive (nur SSH) — SO_KEEPALIVE auf dem Socket.

Während der Wiederverbindung: Die Statusleiste zeigt den Fortschritt; der Scrollback bleibt erhalten; jeder Bereich hat seinen eigenen unabhängigen Wiederverbindungsstatus.

Schaltflächen

Bis zu 20 benutzerdefinierte Aktionsschaltflächen in der unteren Leiste. Klicken zum Senden, Rechtsklick zum Bearbeiten.

Aktionszeichenfolgensyntax

EscapeBedeutung
\rWagenrücklauf (Enter)
\nZeilenvorschub
\tTabulator
\bRücktaste
\eEscape (0x1B)
\p1 Sekunde pausieren
\\Literaler Backslash

Hinweis: Unbekannte Escape-Sequenzen (z. B. \q) werden stillschweigend verworfen und als Warnung protokolliert. Nur die oben aufgeführten Escapes werden verarbeitet.

Beispiele

1. Einen Befehl senden: uptime\r

2. Serielle Anmeldung: \r\padmin\rs3cret\r (Enter → 1 s → admin+Enter → Passwort+Enter)

3. Strg-C senden: \x03

4. ESC dann diagnostizieren: \e\pshow interfaces\r

Schleife und Markierungsschaltflächen

  • Schleife — Rechtsklick auf eine Benutzerschaltfläche → Diese Schaltfläche wiederholen, um in einem Intervall zu wiederholen.
  • Markierung — einen Zeitstempelmarker in das Sitzungsprotokoll einfügen. Linksklick = markieren, Rechtsklick = Bezeichnung bearbeiten.

Snapshots

Ein Snapshot ist eine benannte Befehlsfolge, die auf einer Sitzung ausgeführt wird, Ausgaben erfasst und eine Datei mit Zeitstempel schreibt. Optional wird das Ergebnis per E-Mail versendet.

Konfigurieren

Snapshots → Snapshots verwalten…

  • Name / Beschreibung
  • Befehle — jede Zeile: Befehlszeichenfolge, Verzögerung (ms), optionaler Erwartungsprompt (Regex), maximale Wartezeit (ms).
  • Empfänger, Betreffvorlage ({session}, {snapshot}, {timestamp}).
  • E-Mail senden / Datei anhängen.

Die Erwartungsprompt-Regex wird gegen ein rollendes 64-KB-Fenster der Schrittausgabe abgeglichen, was verhindert, dass pathologische Muster die GUI blockieren.

Ausführen

  • Menü: Snapshots → <Name> oder Rechtsklick auf einen Tab → Snapshot ausführen.
  • Dateien: Log-Verzeichnis/snapshots/. Durchsuchen über Snapshots → Snapshots durchsuchen….

Automatisierungsregeln

Snapshots → Automatisierungsregeln…

  • on_connect — einmal beim Sitzungsaufbau ausführen.
  • cron — 5-Feld-Cron-Zeitplan.
  • pattern — Regex-Übereinstimmung in der Sitzungsausgabe, mit Abklingzeit (Sekunden).

Trigger, periodische Aktionen, Startaktionen

Konfiguriert in Einstellungen → Präferenzen:

  • Trigger — Muster in der Sitzungsausgabe erkennen und eine Aktion senden. Ratengesteuert: Jeder Trigger hat eine 2-s-Abklingzeit pro Muster und ein globales Limit von 20 Auslösungen pro 10 s über alle Trigger einer Sitzung.
  • Periodische Aktionen — alle N Sekunden eine Aktion senden.
  • Startaktionen — einmal beim Verbinden eine Aktion senden.

SMTP

Einstellungen → SMTP…

  • Host / Port
  • Sicherheit — Keine, STARTTLS oder SMTPS. Wenn ein Benutzername konfiguriert ist, ist STARTTLS oder SMTPS erforderlich — Klartext-Authentifizierung wird abgelehnt, um das Leaken von Zugangsdaten zu verhindern.
  • Benutzername / Passwort — das Passwort wird verschlüsselt gespeichert (DPAPI unter Windows, Keychain unter macOS).
  • Absenderadresse / Name — wird auch als Zieladresse für An Administrator senden verwendet.
  • Timeout (s)

Die TLS-Zertifikatsvalidierung wird erzwungen: selbstsignierte, abgelaufene oder mit nicht übereinstimmendem Hostnamen versehene Zertifikate werden mit einer klaren Fehlermeldung abgelehnt.

Protokolle

  • Protokolle → Protokollordner öffnen — eine Datei mit Zeitstempel pro Sitzung.
  • Snapshot-Aufnahmen: Unterordner snapshots/.
  • Sitzungsspezifischer Protokollierungsschalter im Tab Allgemein des Sitzungseditors.
  • Automatischer Fallback: Wenn in das konfigurierte Protokollverzeichnis nicht geschrieben werden kann (OneDrive-Reparse-Point ohne Anmeldung, Controlled Folder Access oder ACL-Einschränkungen), werden Sitzungsprotokolle stattdessen unter Windows nach C:\TSCRT\sessions\ bzw. unter macOS/Linux nach ~/TSCRT/sessions/ geschrieben. Protokollordner öffnen erkennt dies und öffnet automatisch das Fallback-Verzeichnis, mit einem Hinweis in der Statusleiste.

Sicherheit

SSH-Hostschlüsselverifizierung

TSCRT verwendet ein Trust On First Use (TOFU)-Modell. Bei der ersten Verbindung zu einem Server wird der Hostschlüssel an ~/tscrt/known_hosts (OpenSSH-Format) gebunden und sein SHA-256-Fingerabdruck protokolliert.

  • Übereinstimmung — die Verbindung wird stillschweigend fortgesetzt.
  • Neuer Host — der Schlüssel wird automatisch gebunden und persistiert.
  • Abweichung — die Verbindung wird abgelehnt mit einer Fehlermeldung, die den Server-Fingerabdruck und den known_hosts-Pfad anzeigt. Dies schützt vor Man-in-the-Middle-Angriffen. Wenn der Server-Schlüssel sich tatsächlich geändert hat, entfernen Sie den veralteten Eintrag aus der known_hosts-Datei und verbinden Sie sich erneut.

Speicherung von Zugangsdaten

  • Windows — SSH- und SMTP-Passwörter werden mit DPAPI verschlüsselt, gebunden an das aktuelle Benutzerkonto. Die Profildatei wird zusätzlich durch eine ACL nur für den Eigentümer geschützt.
  • macOS — Passwörter werden im System-Keychain gespeichert (über Security.framework). Das Profil enthält nur eine Referenz keychain:<uuid>.
  • Linux — Passwörter werden derzeit im Profil als Klartext gespeichert (0600-Berechtigungen). Eine zukünftige Version könnte libsecret integrieren.
  • Von Windows nach macOS migrierte Profile verlieren verschlüsselte Passwörter. Geben Sie diese nach der Migration erneut ein.

Absturzdumps

Wenn TSCRT abstürzt, wird ein Dump in ~/tscrt/dumps/ geschrieben, mit nur für den Eigentümer zugänglichen Berechtigungen (0600 unter POSIX, ACL nur für den Eigentümer unter Windows). Dumps, die älter als 7 Tage sind, werden beim Start automatisch gelöscht.

Support & Diagnose

Hilfe → Diagnose exportieren… packt eine gzip-komprimierte Textdatei (tscrt-diag-YYYYMMDD-HHMMSS.txt.gz), die Systeminformationen, aufgelöste Pfade, ausgewählte Umgebungsvariablen, eine Schreibbarkeitsprüfung, aktuelle App-Log-Dateien (letzte 7 Tage) und die Namen etwaiger Absturzdumps enthält. Die typische Größe liegt unter 1 MB. Die maximale Größe vor der Komprimierung ist auf 10 MB begrenzt — ältere Log-Inhalte werden bei Bedarf mit einem ausdrücklichen Marker abgeschnitten.

Ein Einwilligungsdialog zeigt den exakten Inhalt an, bevor das Bundle geschrieben wird. Das Bundle enthält nicht: Passwörter, private SSH-Schlüssel, das Sitzungsprofil (Hosts, Snapshots, Automatisierungsregeln), Snapshot-Dateien, sitzungsspezifische Terminal-Aufzeichnungen oder die eingegebenen Befehle.

Überprüfen Sie die generierte .txt.gz, bevor Sie sie mit dem Support teilen.

Profil importieren / exportieren

  • Datei → Profil exportieren… — vollständig oder teilweise.
  • Datei → Profil importieren… — wird in das aktuelle Profil gemischt; Konflikte werden gemeldet.
