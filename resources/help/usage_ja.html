TSCRT 使い方ガイド

TSCRT は SSH2 およびシリアルコンソール用のターミナルエミュレーターです。主要な機能を章ごとに解説します。

  • セッション
  • タブ
  • 分割ペイン / 入力ブロードキャスト / 自動再接続
  • ボタン（例付き）
  • スナップショット（例付き）
  • 自動化ルール
  • トリガー / 定期実行 / 起動時アクション
  • SMTP（メール送信）
  • ログ
  • プロファイルのインポート / エクスポート
  • キーボードショートカット

セッション

セッションとは、SSH あるいはシリアル接続の定義です。

  • Sessions → Manage sessions… でセッションを追加・編集・複製・削除できます。左ドックのツリーでフォルダ別に整理されます。
  • SSH はホスト、ポート、ユーザ名、パスワードまたは秘密鍵ファイルが必要です。
  • シリアル はデバイス名（Windows の COM3、Linux/macOS の /dev/ttyUSB0 など）、ボーレート、データビット、ストップビット、パリティ、フロー制御が必要です。
  • 左ドックのセッションをダブルクリックすると新しいタブで開きます。

タブ

  • 接続ごとにタブが 1 つ割り当てられます。タブ名の横にある小さな × でそのタブだけを閉じます。
  • F11 でフルスクリーン モードを切り替え（ドックとメニューバーを隠す）。
  • セッションごとにコマンド入力欄とボタンバーの表示／非表示をセッション エディターで指定できます。

分割ペイン / 入力ブロードキャスト / 自動再接続

分割ペイン

1 つのタブ内でターミナルを横／縦に分割し、複数のセッションを同時に扱えます。分割時はアクティブ ペインと同じプロファイルでセッションが新規に作られるので、同じホストに複数のシェルを開いたり、片方でログを監視しながらもう片方でコマンドを入力する用途に向きます。

  • Ctrl+Shift+H — 横分割（左右）
  • Ctrl+Shift+V — 縦分割（上下）
  • Ctrl+Shift+W — アクティブ ペインを閉じる。最後のペインを閉じるとタブ自体が閉じます。
  • 分割は再帰的です。横分割の片側を縦に分割すれば格子状のレイアウトになります。
  • アクティブ ペインは 1px の青い枠線で表示されます。マウス クリックやキーボード フォーカスの移動で自動的に切り替わります。
  • ペイン間の境界線をドラッグしてサイズを調整できます。
  • FindBar / Mark / Loop / コマンド入力欄は常に アクティブ ペインに対して動作します。

入力ブロードキャスト

複数のサーバーを同時に管理する際、同じコマンドを 1 度入力するだけですべてのペインに送信します。

  • View → Broadcast Input または Ctrl+Shift+B で切り替えます。
  • 有効化すると、現在のタブのすべてのペインが オレンジの枠線になります。
  • キー入力、貼り付け（Ctrl+V）、IME による日本語入力、マウスレポートが他のすべてのペインのセッションへ同時に送信されます。
  • 切り替えは タブ単位です。タブを切り替えるとメニューのチェック表示もそのタブの状態に更新されます。
  • 注意: 異なるシェル／デバイスに同じキーを送ると意図しない結果になる可能性があるため、同じ OS／同じディレクトリ／同じシェルなど揃った環境で使うのが安全です。

自動再接続 + Keepalive

ネットワークが一時的に切断された際に自動で再接続し、NAT／ファイアウォール 越しの無応答切断を防ぐ機能です。

設定場所: Sessions → Manage sessions… → セッション編集ダイアログ下部の Advanced グループ

  • Auto reconnect — 切断時に自動再接続を有効化します。ユーザーが手動で閉じたタブは再接続しません。
  • Max attempts — 最大再試行回数。0（unlimited）で無制限。
  • Base delay (ms) — 最初の再試行までの待機時間。以後は指数バックオフ（2 倍ずつ）で増加し、30 秒で頭打ちになります。0〜250 ms のランダム ジッターが加わるので一斉再試行を回避できます。
    例（base 500）: 500 ms → 1 s → 2 s → 4 s → 8 s → 16 s → 30 s → 30 s …
  • SSH keepalive (s)（SSH のみ）— libssh2 keepalive の間隔。0 = off。推奨は 30 s。SSH プロトコル レベルで定期的に ping を送信し、NAT セッションの期限切れやステートフル ファイアウォールによる切断を防ぎます。
  • TCP keepalive（SSH のみ）— TCP ソケットに SO_KEEPALIVE を設定。OS レベルでデッド リンクを検出するので、SSH keepalive と併用するとより速く検出できます。

再接続中の挙動:

  • 再接続が始まるとステータス バーに "Reconnecting <session> in N s (attempt M/Max)..." が表示されます。
  • 再接続が成功するとスクロールバックは保持されます — 同じ TerminalWidget に新しいセッションを再アタッチする方式なので、画面の履歴は失われません。
  • 再試行回数の上限に達すると "Reconnect failed for <session>: ..." が表示され、以後試行しません。
  • シリアル セッションも自動再接続に対応しますが、同じデバイス パス（COM3 など）に限られます。USB 抜き差しでポート番号が変わると失敗します。
  • 分割タブ内の各ペインは 独立した再接続状態を持ちます。1 つのペインが再試行中でも他のペインには影響しません。

ボタン

画面下部のボタンバーには最大 20 個のユーザー定義ボタンを配置できます。ボタンをクリックすると、設定されたアクション文字列がアクティブなセッションへ送信されます。ボタンを右クリックでスロットを編集できます。

アクション文字列の文法

エスケープ意味
\rキャリッジリターン（Enter）
\nラインフィード
\tタブ
\bバックスペース
\eESC (0x1B)
\p1 秒待機
\\バックスラッシュ リテラル

例

1. 単一コマンドの送信 — ラベル uptime、アクション:

uptime\r

uptime と入力して Enter を押します。

2. シリアルコンソールのログイン自動化 — ラベル login、アクション:

\r\padmin\rs3cret\r

Enter → 1 秒待機 → admin+Enter → s3cret+Enter の順で送信します。

3. Ctrl キーの送信 — ラベル ^C、アクション:

\x03

\xNN の 16 進エスケープで任意のバイトを挿入できます。

4. キャンセル後に診断コマンドを実行 — ラベル diag、アクション:

\e\pshow interfaces\r

ESC 送信 → 1 秒待機 → show interfaces を実行。

Loop / Mark 特殊ボタン

  • Loop — ユーザー ボタンを右クリックして Loop this button を選ぶと、そのアクションを一定間隔で繰り返します。毎ティックごとに Loop ボタンが点滅し、Loop を再度クリックすると停止します。
  • Mark — セッション ログにタイムスタンプ付きマーカー行を挿入します。左クリックで即座にマーク、右クリックでラベル編集。

スナップショット

スナップショットは、名前の付いたコマンド列をセッションで順次実行し、その出力をタイムスタンプ付きファイルとして保存する機能です。オプションでメール送信もできます。

スナップショットの設定

Snapshots → Manage snapshots… で以下の項目を設定します。

  • 名前 / 説明 — Snapshots メニューに表示されます。
  • Commands — 順序付きリスト。各行に次の項目があります。
    • Command: アクション文字列（ボタンと同じ文法。例: show version\r）
    • Delay (ms): 送信後の最小待機時間（デバイスの応答時間）
    • Expect prompt: 正規表現 — マッチすればその段階を早期終了
    • Max wait (ms): Expect prompt 使用時の最大待機時間
  • Recipients — 1 行に 1 つの宛先メールアドレス。
  • Subject template — {session}、{snapshot}、{timestamp} のプレースホルダーが使えます。
  • Send email / Attach file — メール送信の有無と、本文埋め込み vs 添付の選択。

実行方法

  • Snapshots → <スナップショット名> メニューでアクティブなセッションに対して実行します。
  • 結果ファイルは log dir/snapshots/ 以下に保存されます。Snapshots → Open snapshot folder でエクスプローラーを開くか、Snapshots → Browse snapshots… で TSCRT 内蔵ブラウザを開けます。
  • 内蔵ブラウザでファイルを右クリックし Send to administrator を選ぶと、SMTP の From アドレス（メール サーバーに登録された管理者アカウント）へファイルを添付して送信します。

例

1. Linux ヘルスチェック

name         : linux-health
description  : 負荷 / ディスク / メモリ スナップショット
command 1    : uptime\r               delay: 500
command 2    : df -h\r                delay: 500
command 3    : free -m\r              delay: 500
command 4    : dmesg | tail -n 20\r   delay: 800
subject_tmpl : [TSCRT] {session} health {timestamp}
recipients   : ops@example.com
send_email   : yes
attach_file  : yes

2. Cisco スイッチの設定バックアップ — expect_prompt でページング回避

name         : switch-config
command 1    : terminal length 0\r    delay: 200
command 2    : show running-config\r  delay: 0
               expect_prompt: #\s*$   max_wait: 15000
command 3    : show version\r         delay: 0
               expect_prompt: #\s*$   max_wait: 5000
subject_tmpl : config-backup {session} {timestamp}

正規表現 #\s*$ は行末のスイッチ プロンプトに一致するため、出力が終わった瞬間に次のステップへ進みます。

3. 多段の待機を含む例

command 1: \e             delay: 300    (ESC 送信)
command 2: menu\r         delay: 500
command 3: \p4\r          delay: 1000   (1 秒待機 → "4" → Enter)

自動化ルール

Snapshots → Automation rules… でスナップショットをトリガーに紐付けます。3 種類あります。

  • on_connect — セッション接続時に 1 回実行。
  • cron — 5 フィールド cron（m h dom mon dow）。例: */15 * * * * は 15 分ごと。
  • pattern — セッションからの受信データが正規表現に一致したときに実行。秒単位のクールダウンで過剰発火を防ぎます。

ルールは特定セッション名に限定することも、空欄にして全セッションに適用することもできます。

トリガー / 定期実行 / 起動時アクション

Settings → Preferences で設定します。

  • Triggers — セッション出力が正規表現に一致したときアクション文字列を送信します。プロンプト自動応答などに便利。
  • Periodics — 指定セッションに N 秒ごとにアクションを送信します（keepalive、ポーリングなど）。
  • Startup actions — セッション接続直後に 1 度だけアクションを送信します。

SMTP

Settings → SMTP… で、スナップショットのメール送信と Send to administrator が使用する送信サーバーを設定します。

  • Host / Port — 例: smtp.gmail.com / 587。
  • Security — None、STARTTLS、SMTPS（暗黙 TLS）から選択。
  • Username / Password — パスワードは暗号化されて保存されます。
  • From address / name — From ヘッダーに使用され、このアドレスが「管理者」アドレスとして Send to administrator で利用されます。
  • Timeout (s) — ソケット操作ごとのタイムアウト。

ログ

  • Logs → Open log folder でログ ディレクトリを開きます。セッションごとにタイムスタンプ付きファイルが作成されます。
  • スナップショットの結果ファイルは snapshots/ サブフォルダに保存されます。
  • セッションごとのログ有効化はセッション エディターの Log enabled で切り替えます。

プロファイルのインポート / エクスポート

プロファイルにはセッション、ボタン、スナップショット、ルール、SMTP 設定が含まれます。

  • File → Export profile… — 全体または一部のセクションのみを選択してエクスポート。
  • File → Import profile… — 現在のプロファイルにマージ。競合があれば報告されます。

  • Ctrl+Shift+V — アクティブ ペインを縦分割
