TSCRT 使用ガイド

TSCRTはSSH2およびシリアルコンソール用のターミナルエミュレータです。主要な機能へのリンク:

  • セッション
  • タブ & マルチウィンドウ
  • 分割ペイン / 入力ブロードキャスト / 自動再接続
  • ボタン（例付き）
  • スナップショット（例付き）
  • 自動化ルール
  • トリガー / 定期実行 / 起動アクション
  • SMTP（メール送信）
  • ログ
  • セキュリティ
  • プロファイルのインポート / エクスポート
  • キーボードショートカット

セッション

セッションはSSHまたはシリアル接続の定義です。

  • Sessions → Manage sessions… — セッションの追加・編集・複製・削除。左ドックのツリーに整理されます。
  • SSH: ホスト、ポート、ユーザー名、パスワードまたは秘密鍵ファイル。
  • シリアル: デバイス名（例: COM3）、ボーレート、データ/ストップ/パリティビット、フロー制御。

セッションエディタ（3タブ構成）

  • Connection — 接続パラメータ。
  • General — セッション名、接続タイプ、セッションごとの端末タイプ（空 = グローバルデフォルト使用）、ログ設定、全画面表示オプション。
  • Advanced — 自動再接続設定、SSH keepalive。

タブ & マルチウィンドウ

  • Ctrl+Alt+← / → — 前 / 次のタブ。
  • F11 — 全画面切替。すべてのショートカットは全画面でも動作します。

タブ右クリックメニュー

  • Rename / Duplicate / Run Snapshot / Pin
  • Detach to New Window — タブを新しいTSCRTウィンドウに分離（接続維持）。

ウィンドウ間タブドラッグ

タブをドラッグして別のTSCRTウィンドウにドロップ、またはウィンドウ外にドロップで新規ウィンドウ作成。接続は維持されます。

分割ペイン / 入力ブロードキャスト / 自動再接続

分割ペイン

  • Ctrl+Shift+H — 水平分割。Ctrl+Shift+V — 垂直分割。
  • Ctrl+Shift+W — アクティブペインを閉じる。
  • アクティブペインは青色ボーダーで表示。

入力ブロードキャスト

  • Ctrl+Shift+Bで切替。全ペインにオレンジ色ボーダー。タブ単位。

自動再接続

  • Max attempts: ハード上限100回。Base delay: 最小250 ms、指数バックオフ、上限30 s。
  • スクロールバックは維持されます。各ペインは独立した再接続状態を持ちます。

ボタン

最大20個。アクション文字列: \r=Enter, \n=LF, \t=Tab, \b=BS, \e=ESC, \p=1秒待機, \\=\。未定義のエスケープは無視されます。

Loop: ボタンを右クリック → 一定間隔で繰り返し。Mark: ログにマーカー挿入。

スナップショット

Snapshots → Manage snapshots…で設定。expect_promptは64 KBローリングウィンドウでマッチング。

実行: メニューまたはタブ右クリック → Run Snapshot。

自動化ルール

on_connect / cron（5フィールド） / pattern（正規表現+クールダウン）。

トリガー / 定期実行 / 起動アクション

Settings → Preferences。トリガーはレート制限付き: パターンごと2秒、セッションごと10秒に最大20回。

SMTP

  • ユーザー名設定時はSTARTTLS/SMTPS必須（平文認証拒否）。
  • パスワードは暗号化保存（Windows: DPAPI、macOS: Keychain）。
  • TLS証明書検証は強制。

ログ

Logs → Open log folder。スナップショット: snapshots/。ログ記録のオン/オフはGeneralタブ。

セキュリティ

SSHホスト鍵検証（TOFU）

初回接続で~/tscrt/known_hostsに固定。不一致時は接続拒否（中間者攻撃防御）。

認証情報

Windows: DPAPI + owner-only ACL。macOS: キーチェーン。Linux: 平文（0600）。

クラッシュダンプ

owner-only権限。7日超で自動削除。

プロファイル

File → Export/Import profile…
