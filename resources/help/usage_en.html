TSCRT Usage Guide

Sessions

A session is a connection definition — either SSH or serial.

  • Sessions → Manage sessions… — add, edit, duplicate, or delete sessions. Sessions are grouped in a tree in the left dock.
  • SSH requires host, port, username, and either a password or a private-key file.
  • Serial requires a device name (e.g. COM3 on Windows, /dev/ttyUSB0 on Linux/macOS), baud rate, data bits, stop bits, parity, and flow control.
  • Double-click a session in the left dock to open it in a new tab.

Tabs

  • Each open connection lives in its own tab. The tab title shows the session name; a small × next to the name closes just that tab.
  • A tab can run in fullscreen mode (hides dock and menu bar) — toggle with F11.
  • Per session you can hide/show the command line widget and the button bar from the session editor.

Buttons

The bottom button bar holds up to 20 user-defined action buttons. Clicking a button sends its action string to the active session. Right-click a button to edit the slot.

Action string syntax

An action string is plain text plus escape sequences:

EscapeMeaning
\rCarriage return (Enter)
\nLine feed
\tTab
\bBackspace
\eEscape (0x1B)
\pPause 1 second before continuing
\\Literal backslash

Examples

1. Send a single command — label uptime, action:

uptime\r

Types uptime and presses Enter.

2. Multi-step login on a serial console — label login, action:

\r\padmin\rs3cret\r

Sends Enter, waits 1 s, sends admin+Enter, then s3cret+Enter.

3. Send a Ctrl key — label ^C, action:

\x03

Arbitrary bytes can be embedded with \xNN hex escapes.

4. Cancel, then run a diagnostic — label diag, action:

\e\pshow interfaces\r

Sends ESC, pauses 1 s, then runs show interfaces.

Loop and mark buttons

Two special buttons sit to the right of the user buttons:

  • Loop — right-click a user button and choose Loop this button to repeat its action on an interval. The Loop button flashes in sync with each tick; click Loop to stop.
  • Mark — inserts a timestamped marker line into the session log, handy when reviewing captures later. Left click = mark now, right click = edit label.

Snapshots

A snapshot is a named sequence of commands that TSCRT runs on a session, captures the output of, and writes to a timestamped file under the snapshot folder. Optionally it can email the result.

Configuring a snapshot

Open Snapshots → Manage snapshots…. For each snapshot, configure:

  • Name / description — shown in the Snapshots menu.
  • Commands — ordered list. Each row has:
    • Command: action string (same syntax as buttons, e.g. show version\r)
    • Delay (ms): minimum wait after sending, to let the device respond
    • Expect prompt: optional regex that ends the step early once matched
    • Max wait (ms): upper bound when Expect prompt is set
  • Recipients — one email address per line.
  • Subject template — may use {session}, {snapshot}, {timestamp} placeholders.
  • Send email / Attach file — whether to mail the result and whether the capture is attached or pasted in the body.

Running snapshots

  • Menu: Snapshots → <snapshot name> runs it on the currently focused session.
  • Output files land under log dir/snapshots/. Open that folder with Snapshots → Open snapshot folder, or browse them inside TSCRT via Snapshots → Browse snapshots….
  • Inside the browser, right-click a file to Send to administrator — the file is attached and mailed to the SMTP From address (your configured admin account).

Examples

1. Linux health check

name         : linux-health
description  : Load / disk / memory snapshot
command 1    : uptime\r               delay: 500
command 2    : df -h\r                delay: 500
command 3    : free -m\r              delay: 500
command 4    : dmesg | tail -n 20\r   delay: 800
subject_tmpl : [TSCRT] {session} health {timestamp}
recipients   : ops@example.com
send_email   : yes
attach_file  : yes

2. Cisco-style switch config grab, using expect_prompt to avoid paging

name         : switch-config
command 1    : terminal length 0\r    delay: 200
command 2    : show running-config\r  delay: 0
               expect_prompt: #\s*$   max_wait: 15000
command 3    : show version\r        delay: 0
               expect_prompt: #\s*$   max_wait: 5000
subject_tmpl : config-backup {session} {timestamp}

The regex #\s*$ matches the switch prompt at the end of a line so the step ends the moment output is complete.

3. Multi-step with pauses

command 1: \e             delay: 300    (send ESC)
command 2: menu\r         delay: 500
command 3: \p4\r          delay: 1000  (pause 1 s, then "4", then Enter)

Automation rules

Snapshots → Automation rules… ties snapshots to triggers. Three kinds:

  • on_connect — runs the named snapshot once as soon as the session connects.
  • cron — standard 5-field cron (m h dom mon dow). Example: */15 * * * * runs every 15 minutes.
  • pattern — runs the snapshot when the session's incoming bytes match a regex, throttled by a cooldown in seconds.

Rules can be scoped to a specific session name, or left blank to apply to all sessions.

Triggers, periodics, startup actions

Configured in Settings → Preferences:

  • Triggers — when a session's output matches a regex, send an action string. Useful for auto-ack prompts.
  • Periodics — send an action every N seconds to a named session (keepalives, polling).
  • Startup actions — send an action once right after a session connects.

SMTP

Settings → SMTP… configures outgoing mail used by snapshots and the Send to administrator action.

  • Host / port — e.g. smtp.gmail.com / 587.
  • Security — None, STARTTLS, or SMTPS (implicit TLS).
  • Username / password — password is encrypted at rest.
  • From address / name — appears in the From header. This is also the "administrator" address used by Send to administrator.
  • Timeout (s) — per-step socket timeout.

Logs

  • Logs → Open log folder — session log files live here, one timestamped file per session.
  • Snapshot captures land in the snapshots/ subfolder.
  • Per-session logging is enabled in the session editor (Log enabled).

Profile import / export

The profile holds sessions, buttons, snapshots, rules, and SMTP settings.

  • File → Export profile… — full or partial (pick which sections to include).
  • File → Import profile… — merges into the current profile; conflicts are reported.

Keyboard shortcuts

  • Ctrl+T — new tab from the current session
  • Ctrl+W — close current tab
  • F11 — toggle fullscreen
  • Ctrl+L — focus command line
  • Ctrl+Shift+S — manage snapshots
