Guia de uso do TSCRT

O TSCRT é um emulador de terminal para SSH2 e consoles seriais. Este guia percorre todos os recursos principais. Ir para uma seção:

Sessões

Uma sessão é uma definição de conexão — SSH ou serial.

  • Sessões → Gerenciar sessões… — adicionar, editar, duplicar ou excluir sessões. As sessões são agrupadas em uma árvore no painel esquerdo.
  • SSH requer host, porta, nome de usuário e uma senha ou um arquivo de chave privada.
  • Serial requer um nome de dispositivo (por exemplo, COM3 no Windows, /dev/cu.usbserial-xxx no macOS), taxa de transmissão, bits de dados/parada/paridade e controle de fluxo.
  • Clique duas vezes em uma sessão no painel esquerdo para abri-la em uma nova aba.

Editor de sessões (três abas)

O editor de sessões é organizado em três abas:

  • Conexão — host/porta/usuário/senha/chave (SSH) ou dispositivo/baud/configurações de linha (serial). Esta aba abre primeiro para que os campos alterados com mais frequência fiquem imediatamente visíveis.
  • Geral — nome da sessão, tipo de conexão, tipo de terminal por sessão (por exemplo, xterm-256color, vt100), alternância de registro de sessão e opções de exibição em tela cheia. Se o tipo de terminal for deixado em branco, será usado o padrão global de Configurações → Preferências.
  • Avançado — configurações de reconexão automática (tentativas máximas, atraso base) e keepalive SSH (intervalo, keepalive TCP). O grupo keepalive fica oculto para sessões seriais.

Abas e multijanela

  • Cada conexão aberta vive em sua própria aba. O título da aba mostra o nome da sessão; um pequeno × ao lado do nome fecha apenas essa aba.
  • Ctrl+Alt+← / → — mover para a aba anterior / próxima.
  • F11 — alternar tela cheia (oculta painel e barra de menu). Todos os atalhos continuam funcionando em tela cheia.

Menu de contexto da aba (clique direito)

Clique com o botão direito no título de uma aba para acessar:

  • Renomear — alterar o título exibido da aba.
  • Duplicar — abrir a mesma sessão em uma nova aba.
  • Executar snapshot → — executar um snapshot na sessão desta aba.
  • Fixar / Desafixar — as abas fixadas são restauradas automaticamente na próxima inicialização.
  • Destacar para nova janela — mover a aba para uma janela TSCRT totalmente nova. A sessão SSH/serial permanece conectada — sem desconexão/reconexão. Desabilitado quando há apenas uma aba.
  • Vincular aba à aba par… — emparelhar esta aba com uma aba em outra instância do TSCRT na mesma máquina. Ativar qualquer uma das abas também alterna a aba par na outra janela. Útil para layouts lado a lado (por exemplo, console no monitor esquerdo, visualizador de logs no direito).
  • Desvincular aba — aparece no lugar de Vincular quando a aba já tem um par; remove o emparelhamento.
  • Fechar — fechar a aba.

Arrastar abas entre janelas

Arraste uma aba da barra de abas e solte-a em outra janela do TSCRT para transferir a sessão. Se você soltá-la fora de qualquer janela do TSCRT, uma nova janela é criada automaticamente. A conexão ativa é preservada durante todo o processo.

Vínculos de abas entre instâncias

Os vínculos de abas são armazenados no perfil e se reconectam automaticamente na próxima vez que ambas as instâncias do TSCRT estiverem em execução. Os vínculos sobrevivem a reinicializações. A descoberta de pares ocorre por um canal IPC estritamente local; nenhum tráfego de rede é gerado.

Painéis divididos / Difusão de entrada / Reconexão automática

Painéis divididos

Uma única aba pode conter vários painéis de terminal dispostos em uma árvore divisória recursiva. A divisão reutiliza o perfil do painel ativo, de modo que o novo painel aterrissa no mesmo host.

  • Ctrl+Shift+H — dividir horizontalmente (lado a lado).
  • Ctrl+Shift+V — dividir verticalmente (em cima / embaixo).
  • Ctrl+Shift+W — fechar o painel ativo. Fechar o último painel fecha a aba.
  • As divisões são recursivas: uma divisão horizontal seguida de uma vertical produz uma grade.
  • O painel ativo é marcado com uma borda azul de 1 px. Cliques do mouse e foco do teclado o atualizam automaticamente.
  • Arraste a alça do divisor para redimensionar.
  • Barra de busca, Marcar, Loop e a linha de comando sempre atuam no painel ativo.

Difusão de entrada

Espelhe as teclas digitadas em vários painéis — útil para administrar uma frota de servidores.

  • Alterne com Visualizar → Difundir entrada ou Ctrl+Shift+B.
  • Quando habilitado, cada painel ganha uma borda laranja.
  • Teclas digitadas, colagem, entrada de IME e relatórios do mouse de um painel são encaminhados para todos os outros painéis.
  • A alternância é por aba.

Reconexão automática e keepalive SSH

Configurados na aba Avançado do editor de sessões.

  • Reconexão automática — restaurar a sessão após uma queda. Abas fechadas manualmente não serão reconectadas.
  • Tentativas máximas — limite de desistência. 0 = ilimitado (teto rígido em 100).
  • Atraso base (ms) — backoff inicial (mínimo 250 ms). Cada nova tentativa dobra, teto de 30 s, com jitter de 0–250 ms.
    Exemplo com base 500: 500 ms → 1 s → 2 s → 4 s → 8 s → 16 s → 30 s …
  • Keepalive SSH (s) (somente SSH) — intervalo de keepalive do libssh2. Recomendado 30 s.
  • Keepalive TCP (somente SSH) — SO_KEEPALIVE no soquete.

Durante a reconexão: a barra de status mostra o progresso; o buffer de rolagem é preservado; cada painel tem seu próprio estado de reconexão independente.

Botões

Até 20 botões de ação definidos pelo usuário na barra inferior. Clique para enviar, clique direito para editar.

Sintaxe da string de ação

EscapeSignificado
\rRetorno de carro (Enter)
\nAvanço de linha
\tTabulação
\bRetrocesso
\eEscape (0x1B)
\pPausa de 1 segundo
\\Barra invertida literal

Observação: Sequências de escape desconhecidas (por exemplo, \q) são descartadas silenciosamente e registradas como aviso. Apenas os escapes listados acima são processados.

Exemplos

1. Enviar um comando: uptime\r

2. Login serial: \r\padmin\rs3cret\r (Enter → 1 s → admin+Enter → senha+Enter)

3. Enviar Ctrl-C: \x03

4. ESC e então diagnosticar: \e\pshow interfaces\r

Botões Loop e Marca

  • Loop — clique direito em um botão de usuário → Repetir este botão para repetir em um intervalo.
  • Marca — inserir um marcador com timestamp no log da sessão. Clique esquerdo = marcar, clique direito = editar rótulo.

Snapshots

Um snapshot é uma sequência nomeada de comandos que é executada em uma sessão, captura a saída e grava um arquivo com timestamp. Opcionalmente envia o resultado por e-mail.

Configuração

Snapshots → Gerenciar snapshots…

  • Nome / descrição
  • Comandos — cada linha: string do comando, atraso (ms), prompt esperado opcional (regex), espera máxima (ms).
  • Destinatários, modelo de assunto ({session}, {snapshot}, {timestamp}).
  • Enviar e-mail / Anexar arquivo.

O regex do prompt esperado é comparado com uma janela rolante de 64 KB da saída da etapa, o que impede que padrões patológicos travem a GUI.

Execução

  • Menu: Snapshots → <nome>, ou clique direito em uma aba → Executar snapshot.
  • Arquivos: diretório de logs/snapshots/. Navegue via Snapshots → Navegar snapshots….

Regras de automação

Snapshots → Regras de automação…

  • on_connect — executar uma vez ao conectar a sessão.
  • cron — agendamento cron de 5 campos.
  • pattern — correspondência regex na saída da sessão, com resfriamento (segundos).

Gatilhos, periódicos, ações de inicialização

Configurados em Configurações → Preferências:

  • Gatilhos — detectam padrões na saída da sessão e enviam uma ação. Com limite de taxa: cada gatilho tem um resfriamento de 2 s por padrão e um teto global de 20 disparos a cada 10 s em todos os gatilhos de uma sessão.
  • Periódicos — enviam uma ação a cada N segundos.
  • Ações de inicialização — enviam uma ação uma vez ao conectar.

SMTP

Configurações → SMTP…

  • Host / porta
  • Segurança — Nenhuma, STARTTLS ou SMTPS. Se um nome de usuário estiver configurado, STARTTLS ou SMTPS é obrigatório — a autenticação em texto simples é recusada para evitar vazamento de credenciais.
  • Nome de usuário / senha — a senha é criptografada em repouso (DPAPI no Windows, Keychain no macOS).
  • Endereço / nome do remetente — também usado como endereço de destino para Enviar ao administrador.
  • Timeout (s)

A validação do certificado TLS é aplicada: certificados autoassinados, expirados ou com nome de host incompatível são recusados com uma mensagem de erro clara.

Logs

  • Logs → Abrir pasta de logs — um arquivo com timestamp por sessão.
  • Capturas de snapshot: subpasta snapshots/.
  • Alternância de log por sessão na aba Geral do editor de sessões.
  • Fallback automático: se não for possível gravar no diretório de logs configurado (ponto de reanálise do OneDrive sem login, Controlled Folder Access ou restrições de ACL), os logs de sessão são gravados em C:\TSCRT\sessions\ no Windows, ou ~/TSCRT/sessions/ no macOS/Linux. Abrir pasta de logs detecta isso e abre automaticamente o diretório de fallback, com um aviso na barra de status.

Segurança

Verificação de chave de host SSH

O TSCRT usa um modelo Trust On First Use (TOFU). Na primeira conexão a um servidor, a chave do host é fixada em ~/tscrt/known_hosts (formato OpenSSH) e sua impressão digital SHA-256 é registrada.

  • Correspondência — a conexão prossegue silenciosamente.
  • Novo host — a chave é automaticamente fixada e persistida.
  • Incompatibilidade — a conexão é recusada com um erro mostrando a impressão digital do servidor e o caminho do known_hosts. Isso protege contra ataques man-in-the-middle. Se a chave do servidor mudou legitimamente, remova a entrada obsoleta do arquivo known_hosts e reconecte.

Armazenamento de credenciais

  • Windows — senhas SSH e SMTP são criptografadas com DPAPI, vinculadas à conta de usuário atual. O arquivo de perfil é protegido adicionalmente por uma ACL exclusiva do proprietário.
  • macOS — as senhas são armazenadas no Keychain do sistema (via Security.framework). O perfil contém apenas uma referência keychain:<uuid>.
  • Linux — as senhas atualmente são armazenadas como texto simples no perfil (permissões 0600). Uma versão futura pode integrar o libsecret.
  • Perfis migrados do Windows para o macOS perderão as senhas criptografadas. Insira-as novamente após a migração.

Despejos de falha

Se o TSCRT travar, um despejo é gravado em ~/tscrt/dumps/ com permissões exclusivas do proprietário (0600 no POSIX, ACL exclusiva do proprietário no Windows). Despejos com mais de 7 dias são excluídos automaticamente na inicialização.

Suporte & diagnósticos

Ajuda → Exportar diagnósticos… empacota um arquivo de texto compactado em gzip (tscrt-diag-YYYYMMDD-HHMMSS.txt.gz) contendo informações do sistema, caminhos resolvidos, variáveis de ambiente selecionadas, uma sondagem de gravação, arquivos de log recentes do aplicativo (últimos 7 dias) e os nomes de quaisquer despejos de falha. O tamanho típico é inferior a 1 MB. O tamanho máximo antes da compactação é limitado a 10 MB — o conteúdo de log mais antigo é truncado com um marcador explícito, se necessário.

Uma caixa de diálogo de consentimento mostra o conteúdo exato antes de o pacote ser gravado. O pacote não inclui senhas, chaves privadas SSH, o perfil de sessão (hosts, snapshots, regras de automação), arquivos de snapshot, capturas de terminal por sessão, nem os comandos que você digitou.

Revise o arquivo .txt.gz gerado antes de compartilhá-lo com o suporte.

Importação / exportação de perfil

  • Arquivo → Exportar perfil… — completo ou parcial.
  • Arquivo → Importar perfil… — mesclado no perfil atual; conflitos são relatados.
