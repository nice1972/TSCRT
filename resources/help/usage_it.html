Guida all'uso di TSCRT

TSCRT è un emulatore di terminale per SSH2 e console seriali. Questa guida illustra tutte le funzionalità principali. Vai a una sezione:

Sessioni

Una sessione è una definizione di connessione — SSH o seriale.

  • Sessioni → Gestisci sessioni… — aggiungi, modifica, duplica o elimina sessioni. Le sessioni sono raggruppate in un albero nel pannello di sinistra.
  • SSH richiede host, porta, nome utente e una password o un file di chiave privata.
  • Seriale richiede un nome di dispositivo (es. COM3 su Windows, /dev/cu.usbserial-xxx su macOS), baud rate, bit di dati/stop/parità e controllo di flusso.
  • Fai doppio clic su una sessione nel pannello di sinistra per aprirla in una nuova scheda.

Editor di sessione (tre schede)

L'editor di sessione è organizzato in tre schede:

  • Connessione — host/porta/utente/password/chiave (SSH) o dispositivo/baud/impostazioni di linea (seriale). Questa scheda si apre per prima, così i campi modificati più di frequente sono immediatamente visibili.
  • Generale — nome sessione, tipo di connessione, tipo di terminale per sessione (es. xterm-256color, vt100), interruttore di logging della sessione e opzioni di visualizzazione a schermo intero. Se il tipo di terminale viene lasciato vuoto, viene utilizzato il valore predefinito globale da Impostazioni → Preferenze.
  • Avanzate — impostazioni di riconnessione automatica (tentativi massimi, ritardo base) e keepalive SSH (intervallo, keepalive TCP). Il gruppo keepalive è nascosto per le sessioni seriali.

Schede e multifinestra

  • Ogni connessione aperta vive nella propria scheda. Il titolo della scheda mostra il nome della sessione; una piccola × accanto al nome chiude solo quella scheda.
  • Ctrl+Alt+← / → — passa alla scheda precedente / successiva.
  • F11 — attiva/disattiva schermo intero (nasconde pannello e barra dei menu). Tutte le scorciatoie continuano a funzionare a schermo intero.

Menu contestuale della scheda (tasto destro)

Fai clic destro sul titolo di una scheda per accedere a:

  • Rinomina — cambia il titolo della scheda visualizzato.
  • Duplica — apri la stessa sessione in una nuova scheda.
  • Esegui snapshot → — esegui uno snapshot sulla sessione di questa scheda.
  • Fissa / Sblocca — le schede fissate vengono ripristinate automaticamente al prossimo avvio.
  • Stacca in nuova finestra — sposta la scheda in una nuova finestra TSCRT. La sessione SSH/seriale resta connessa — nessuna disconnessione/riconnessione. Disabilitato quando c'è solo una scheda.
  • Chiudi — chiudi la scheda.

Trascinare schede tra finestre

Trascina una scheda dalla barra delle schede e rilasciala su un'altra finestra TSCRT per trasferire la sessione. Se la rilasci fuori da qualsiasi finestra TSCRT, viene creata automaticamente una nuova finestra. La connessione attiva viene mantenuta per tutto il processo.

Pannelli divisi / Trasmissione input / Riconnessione automatica

Pannelli divisi

Una singola scheda può contenere più pannelli terminale disposti in un albero divisorio ricorsivo. La divisione riutilizza il profilo del pannello attivo, così il nuovo pannello finisce sullo stesso host.

  • Ctrl+Shift+H — dividi orizzontalmente (affiancati).
  • Ctrl+Shift+V — dividi verticalmente (sopra / sotto).
  • Ctrl+Shift+W — chiudi il pannello attivo. Chiudere l'ultimo pannello chiude la scheda.
  • Le divisioni sono ricorsive: una divisione orizzontale seguita da una verticale produce una griglia.
  • Il pannello attivo è contrassegnato da un bordo blu di 1 px. I clic del mouse e il focus della tastiera lo aggiornano automaticamente.
  • Trascina la maniglia del divisore per ridimensionare.
  • Barra di ricerca, Segno, Ciclo e la riga di comando agiscono sempre sul pannello attivo.

Trasmissione input

Replica i tasti premuti su più pannelli — utile per amministrare una flotta di server.

  • Attiva/disattiva con Visualizza → Trasmetti input o Ctrl+Shift+B.
  • Quando abilitata, ogni pannello riceve un bordo arancione.
  • Tasti, incolla, input IME e report del mouse da un pannello vengono inoltrati a tutti gli altri pannelli.
  • L'interruttore è per scheda.

Riconnessione automatica e keepalive SSH

Configurati nella scheda Avanzate dell'editor di sessione.

  • Riconnessione automatica — ripristina la sessione dopo una caduta. Le schede chiuse manualmente non verranno riconnesse.
  • Tentativi massimi — limite di resa. 0 = illimitato (tetto rigido a 100).
  • Ritardo base (ms) — backoff iniziale (minimo 250 ms). Ogni nuovo tentativo raddoppia, soffitto 30 s, con jitter di 0–250 ms.
    Esempio con base 500: 500 ms → 1 s → 2 s → 4 s → 8 s → 16 s → 30 s …
  • Keepalive SSH (s) (solo SSH) — intervallo di keepalive di libssh2. Consigliato 30 s.
  • Keepalive TCP (solo SSH) — SO_KEEPALIVE sul socket.

Durante la riconnessione: la barra di stato mostra l'avanzamento; il buffer di scorrimento viene preservato; ogni pannello ha il proprio stato di riconnessione indipendente.

Pulsanti

Fino a 20 pulsanti di azione definiti dall'utente nella barra inferiore. Clic per inviare, clic destro per modificare.

Sintassi della stringa di azione

EscapeSignificato
\rRitorno a capo (Invio)
\nAvanzamento riga
\tTabulazione
\bBackspace
\eEscape (0x1B)
\pPausa di 1 secondo
\\Backslash letterale

Nota: Le sequenze di escape sconosciute (es. \q) vengono scartate silenziosamente e registrate come avviso. Vengono elaborati solo gli escape elencati sopra.

Esempi

1. Invia un comando: uptime\r

2. Login seriale: \r\padmin\rs3cret\r (Invio → 1 s → admin+Invio → password+Invio)

3. Invia Ctrl-C: \x03

4. ESC poi diagnostica: \e\pshow interfaces\r

Pulsanti Ciclo e Segno

  • Ciclo — clic destro su un pulsante utente → Ripeti questo pulsante per ripetere a intervalli.
  • Segno — inserisci un marcatore con timestamp nel log della sessione. Clic sinistro = segna, clic destro = modifica etichetta.

Snapshot

Uno snapshot è una sequenza di comandi denominata che viene eseguita su una sessione, cattura l'output e scrive un file con timestamp. Facoltativamente invia il risultato per e-mail.

Configurazione

Snapshot → Gestisci snapshot…

  • Nome / descrizione
  • Comandi — ogni riga: stringa del comando, ritardo (ms), prompt atteso opzionale (regex), attesa massima (ms).
  • Destinatari, modello oggetto ({session}, {snapshot}, {timestamp}).
  • Invia e-mail / Allega file.

La regex del prompt atteso viene confrontata con una finestra mobile di 64 KB dell'output del passaggio, il che impedisce a modelli patologici di bloccare la GUI.

Esecuzione

  • Menu: Snapshot → <nome>, o clic destro su una scheda → Esegui snapshot.
  • File: directory log/snapshots/. Sfoglia tramite Snapshot → Sfoglia snapshot….

Regole di automazione

Snapshot → Regole di automazione…

  • on_connect — esegui una volta alla connessione della sessione.
  • cron — pianificazione cron a 5 campi.
  • pattern — corrispondenza regex sull'output della sessione, con raffreddamento (secondi).

Trigger, periodici, azioni di avvio

Configurati in Impostazioni → Preferenze:

  • Trigger — rilevano pattern nell'output della sessione e inviano un'azione. A velocità limitata: ogni trigger ha un raffreddamento di 2 s per pattern e un tetto globale di 20 attivazioni ogni 10 s su tutti i trigger di una sessione.
  • Periodici — inviano un'azione ogni N secondi.
  • Azioni di avvio — inviano un'azione una volta alla connessione.

SMTP

Impostazioni → SMTP…

  • Host / porta
  • Sicurezza — Nessuna, STARTTLS o SMTPS. Se è configurato un nome utente, è richiesto STARTTLS o SMTPS — l'autenticazione in chiaro viene rifiutata per prevenire la fuga di credenziali.
  • Nome utente / password — la password è cifrata a riposo (DPAPI su Windows, Keychain su macOS).
  • Indirizzo / nome mittente — usato anche come indirizzo destinatario per Invia all'amministratore.
  • Timeout (s)

La convalida del certificato TLS è obbligatoria: i certificati autofirmati, scaduti o con nome host non corrispondente vengono rifiutati con un messaggio di errore chiaro.

Log

  • Log → Apri cartella log — un file con timestamp per sessione.
  • Catture snapshot: sottocartella snapshots/.
  • Interruttore di logging per sessione nella scheda Generale dell'editor di sessione.

Sicurezza

Verifica della chiave host SSH

TSCRT utilizza un modello Trust On First Use (TOFU). Alla prima connessione a un server, la chiave host viene fissata in ~/tscrt/known_hosts (formato OpenSSH) e la sua impronta SHA-256 viene registrata.

  • Corrispondenza — la connessione procede silenziosamente.
  • Nuovo host — la chiave viene fissata e resa persistente automaticamente.
  • Non corrispondenza — la connessione viene rifiutata con un errore che mostra l'impronta del server e il percorso di known_hosts. Questo protegge dagli attacchi man-in-the-middle. Se la chiave del server è legittimamente cambiata, rimuovi la voce obsoleta dal file known_hosts e riconnettiti.

Archiviazione credenziali

  • Windows — le password SSH e SMTP sono cifrate con DPAPI, legate all'account utente corrente. Il file di profilo è inoltre protetto da una ACL riservata al proprietario.
  • macOS — le password sono archiviate nel Keychain di sistema (tramite Security.framework). Il profilo contiene solo un riferimento keychain:<uuid>.
  • Linux — le password sono attualmente archiviate in chiaro nel profilo (permessi 0600). Una versione futura potrebbe integrare libsecret.
  • I profili migrati da Windows a macOS perderanno le password cifrate. Reinseriscile dopo la migrazione.

Dump di crash

Se TSCRT si blocca, viene scritto un dump in ~/tscrt/dumps/ con permessi riservati al proprietario (0600 su POSIX, ACL riservata al proprietario su Windows). I dump più vecchi di 7 giorni vengono eliminati automaticamente all'avvio.

Importazione / esportazione profilo

  • File → Esporta profilo… — completo o parziale.
  • File → Importa profilo… — unito al profilo corrente; i conflitti vengono segnalati.
