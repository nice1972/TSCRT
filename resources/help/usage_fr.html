Guide d'utilisation de TSCRT

TSCRT est un émulateur de terminal pour SSH2 et consoles série. Ce guide parcourt toutes les fonctionnalités majeures. Accédez à une section :

Sessions

Une session est une définition de connexion — SSH ou série.

  • Sessions → Gérer les sessions… — ajouter, modifier, dupliquer ou supprimer des sessions. Les sessions sont regroupées dans une arborescence dans le panneau gauche.
  • SSH nécessite un hôte, un port, un nom d'utilisateur et soit un mot de passe, soit un fichier de clé privée.
  • Série nécessite un nom de périphérique (par exemple COM3 sous Windows, /dev/cu.usbserial-xxx sous macOS), une vitesse en bauds, des bits de données/stop/parité, et un contrôle de flux.
  • Double-cliquez sur une session dans le panneau gauche pour l'ouvrir dans un nouvel onglet.

Éditeur de session (trois onglets)

L'éditeur de session est organisé en trois onglets :

  • Connexion — hôte/port/utilisateur/mot de passe/clé (SSH) ou périphérique/baud/paramètres de ligne (série). Cet onglet s'ouvre en premier afin que les champs les plus fréquemment modifiés soient immédiatement visibles.
  • Général — nom de session, type de connexion, type de terminal par session (par exemple xterm-256color, vt100), bascule de journalisation de session, et options d'affichage plein écran. Si le type de terminal est laissé vide, la valeur par défaut globale de Paramètres → Préférences est utilisée.
  • Avancé — paramètres de reconnexion automatique (nombre maximal de tentatives, délai de base) et keepalive SSH (intervalle, keepalive TCP). Le groupe keepalive est masqué pour les sessions série.

Onglets et multi-fenêtres

  • Chaque connexion ouverte vit dans son propre onglet. Le titre de l'onglet affiche le nom de la session ; une petite × à côté du nom ferme uniquement cet onglet.
  • Ctrl+Alt+← / → — passer à l'onglet précédent / suivant.
  • F11 — basculer en plein écran (masque le panneau et la barre de menu). Tous les raccourcis continuent à fonctionner en plein écran.

Menu contextuel d'onglet (clic droit)

Cliquez avec le bouton droit sur un titre d'onglet pour accéder à :

  • Renommer — modifier le titre de l'onglet affiché.
  • Dupliquer — ouvrir la même session dans un nouvel onglet.
  • Exécuter l'instantané → — exécuter un instantané sur la session de cet onglet.
  • Épingler / Désépingler — les onglets épinglés sont automatiquement restaurés au prochain lancement.
  • Détacher dans une nouvelle fenêtre — déplacer l'onglet dans une toute nouvelle fenêtre TSCRT. La session SSH/série reste connectée — pas de déconnexion/reconnexion. Désactivé lorsqu'il n'y a qu'un seul onglet.
  • Lier l'onglet à un onglet pair… — appariez cet onglet avec un onglet d'une autre instance de TSCRT sur la même machine. Activer l'un ou l'autre onglet bascule également l'onglet pair dans l'autre fenêtre. Utile pour les dispositions côte à côte (par exemple, console sur le moniteur gauche, visualiseur de journaux sur le droit).
  • Délier l'onglet — apparaît à la place de Lier lorsque l'onglet a déjà un pair ; supprime l'appariement.
  • Fermer — fermer l'onglet.

Glisser les onglets entre les fenêtres

Faites glisser un onglet depuis la barre d'onglets et déposez-le sur une autre fenêtre TSCRT pour transférer la session. Si vous le déposez en dehors de toute fenêtre TSCRT, une nouvelle fenêtre est automatiquement créée. La connexion active est préservée tout au long.

Liaison d'onglets

La liaison d'onglets synchronise l'activation des onglets entre deux fenêtres TSCRT exécutées sur la même machine. Lorsque vous basculez vers un onglet lié dans une fenêtre, l'onglet apparié dans l'autre fenêtre est automatiquement mis au premier plan. Ceci est conçu pour les workflows à double écran — par exemple, une console d'appareil sur l'écran gauche et un visualiseur de journaux ou un second appareil sur l'écran droit.

Créer un lien

  1. Ouvrez deux instances de TSCRT (la première devient le rôle A, la seconde devient automatiquement le rôle B).
  2. Ouvrez les onglets de session souhaités dans chaque fenêtre.
  3. Cliquez avec le bouton droit sur un onglet dans l'une des fenêtres → Lier l'onglet à un onglet pair…
  4. Dans la boîte de dialogue, l'autre instance TSCRT et ses onglets ouverts sont listés. Les onglets portant le même nom de session sont distingués par un numéro de slot (0, 1, …). Sélectionnez l'onglet pair exact avec lequel vous souhaitez apparier et cliquez sur OK.
  5. Une petite icône de chaîne apparaît sur les deux onglets liés pour confirmer l'appariement.

Vous pouvez créer plusieurs liens — par exemple, lier l'onglet 1↔3 et l'onglet 2↔4 pour deux paires de sessions associées.

Fonctionnement de la synchronisation d'activation

  • Chaque lien est identifié par un identifiant de paire unique. Lorsque vous activez un onglet lié, TSCRT diffuse l'identifiant de paire via un canal IPC local (socket Unix / named pipe Windows) à chaque pair connecté.
  • Le pair récepteur recherche l'identifiant de paire dans sa table de liaisons, trouve l'onglet local correspondant et y bascule automatiquement. La fenêtre du pair est également mise au premier plan.
  • Un garde anti-boucle empêche l'activation de rebondir : lorsqu'un onglet est basculé par un message entrant, les diffusions sortantes sont temporairement supprimées.
  • Les liens sont sensibles au rôle — chaque extrémité enregistre à quel rôle de processus (A ou B) elle appartient, de sorte que les noms de session dupliqués entre instances ne causent jamais de liaisons ambiguës.

Sauvegarde & restauration

  • Les liens sont stockés dans le profil et survivent aux redémarrages. Chaque lien enregistre son identifiant de paire, les noms de session des deux extrémités, l'index de slot et le rôle du processus.
  • Chaque processus sauvegarde également sa liste d'onglets ouverts dans un fichier de disposition (layout_A.lst / layout_B.lst) dans le répertoire du profil.
  • Au prochain lancement, le rôle A rouvre automatiquement ses onglets sauvegardés. Si des liens existent et que layout_B.lst n'est pas vide, un second processus TSCRT (--role=B) est lancé automatiquement, rouvrant les onglets côté B et rétablissant toutes les liaisons — l'ensemble de l'environnement à double fenêtre est restauré en un seul clic.

Délier

Cliquez avec le bouton droit sur un onglet lié → Délier l'onglet (actuellement : …). L'icône de chaîne disparaît et l'appariement est supprimé du profil. Si tous les liens sont supprimés, le prochain lancement ne lancera plus automatiquement un second TSCRT.

Découverte des pairs

Chaque processus TSCRT s'enregistre sous $XDG_RUNTIME_DIR/tscrt/peers/ (Linux) ou le répertoire temporaire équivalent (Windows / macOS). Un minuteur de découverte léger vérifie les nouveaux pairs toutes les 3 secondes. Les entrées périmées de processus plantés sont automatiquement nettoyées lorsque le socket est trouvé inaccessible. Aucun trafic réseau ne quitte la machine locale.

Panneaux divisés / Diffusion d'entrée / Reconnexion automatique

Panneaux divisés

Un onglet unique peut contenir plusieurs panneaux de terminal disposés en arborescence de séparateurs récursive. Le fractionnement réutilise le profil du panneau actif afin que le nouveau panneau atterrisse sur le même hôte.

  • Ctrl+Shift+H — diviser horizontalement (côte à côte).
  • Ctrl+Shift+V — diviser verticalement (haut / bas).
  • Ctrl+Shift+W — fermer le panneau actif. Fermer le dernier panneau ferme l'onglet.
  • Les divisions sont récursives : une division horizontale suivie d'une division verticale produit une grille.
  • Le panneau actif est marqué d'une bordure bleue de 1 px. Les clics de souris et le focus clavier le mettent à jour automatiquement.
  • Faites glisser la poignée du séparateur pour redimensionner.
  • La barre de recherche, Marque, Boucle et la ligne de commande agissent toujours sur le panneau actif.

Diffusion d'entrée

Réplique les frappes sur plusieurs panneaux — utile pour administrer une flotte de serveurs.

  • Basculer avec Affichage → Diffuser l'entrée ou Ctrl+Shift+B.
  • Lorsque activé, chaque panneau reçoit une bordure orange.
  • Les frappes, le collage, les entrées IME et les rapports de souris d'un panneau sont transmis à tous les autres panneaux.
  • La bascule est par onglet.

Reconnexion automatique et keepalive SSH

Configuré dans l'onglet Avancé de l'éditeur de session.

  • Reconnexion automatique — restaurer la session après une coupure. Les onglets fermés manuellement ne seront pas reconnectés.
  • Tentatives maximales — limite d'abandon. 0 = illimité (plafond strict à 100).
  • Délai de base (ms) — temporisation initiale (minimum 250 ms). Chaque nouvelle tentative double, plafond à 30 s, avec gigue de 0–250 ms.
    Exemple avec base 500 : 500 ms → 1 s → 2 s → 4 s → 8 s → 16 s → 30 s …
  • Keepalive SSH (s) (SSH uniquement) — intervalle keepalive libssh2. Recommandé 30 s.
  • Keepalive TCP (SSH uniquement) — SO_KEEPALIVE sur le socket.

Pendant la reconnexion : la barre d'état affiche la progression ; le défilement arrière est préservé ; chaque panneau a son propre état de reconnexion indépendant.

Boutons

Jusqu'à 20 boutons d'action définis par l'utilisateur dans la barre inférieure. Clic pour envoyer, clic droit pour modifier.

Syntaxe de la chaîne d'action

ÉchappementSignification
\rRetour chariot (Entrée)
\nSaut de ligne
\tTabulation
\bRetour arrière
\eÉchap (0x1B)
\pPause d'1 seconde
\\Barre oblique inverse littérale

Remarque : Les séquences d'échappement inconnues (par exemple \q) sont silencieusement ignorées et consignées comme avertissement. Seuls les échappements listés ci-dessus sont traités.

Exemples

1. Envoyer une commande : uptime\r

2. Connexion série : \r\padmin\rs3cret\r (Entrée → 1 s → admin+Entrée → mot de passe+Entrée)

3. Envoyer Ctrl-C : \x03

4. ESC puis diagnostiquer : \e\pshow interfaces\r

Boutons Boucle et Marque

  • Boucle — clic droit sur un bouton utilisateur → Boucler ce bouton pour répéter à un intervalle.
  • Marque — insérer un marqueur horodaté dans le journal de session. Clic gauche = marquer, clic droit = modifier l'étiquette.

Instantanés

Un instantané est une séquence nommée de commandes qui s'exécute sur une session, capture la sortie et écrit un fichier horodaté. Lorsqu'il est configuré, TSCRT envoie automatiquement le résultat capturé par e-mail via le client SMTP intégré.

Configuration

Instantanés → Gérer les instantanés…

  • Nom / description
  • Commandes — chaque ligne : chaîne de commande, délai (ms), invite attendue optionnelle (regex), attente maximale (ms).
  • Destinataires — une ou plusieurs adresses e-mail qui reçoivent automatiquement le résultat de l'instantané.
  • Modèle d'objet — prend en charge les espaces réservés : {session}, {snapshot}, {timestamp}.
  • Envoyer un e-mail — activer/désactiver l'envoi automatique d'e-mail par instantané.
  • Joindre le fichier — lorsqu'il est activé, le fichier de sortie capturé est joint à l'e-mail.

La regex de l'invite attendue est comparée à une fenêtre glissante de 64 Ko de la sortie de l'étape, ce qui empêche les motifs pathologiques de bloquer l'interface graphique.

Envoi automatique d'e-mails

Lorsque Envoyer un e-mail est activé pour un instantané et que SMTP est configuré (voir SMTP), TSCRT envoie automatiquement le résultat de l'instantané à tous les destinataires listés immédiatement après la fin de l'exécution. Combiné avec les règles d'automatisation (on_connect / cron / motif), cela fournit une surveillance entièrement automatisée : TSCRT exécute l'instantané selon un planning ou un déclencheur, capture la sortie et l'envoie par e-mail à l'équipe — sans intervention manuelle.

Exécution

  • Menu : Instantanés → <nom>, ou clic droit sur un onglet → Exécuter l'instantané.
  • Fichiers : répertoire de journaux/snapshots/. Parcourir via Instantanés → Parcourir les instantanés….

Règles d'automatisation

Instantanés → Règles d'automatisation…

  • on_connect — exécuter une fois à la connexion de la session.
  • cron — planification cron à 5 champs.
  • pattern — correspondance regex sur la sortie de session, avec temps de refroidissement (secondes).

Déclencheurs, périodiques, actions de démarrage

Configurés dans Paramètres → Préférences :

  • Déclencheurs — détectent des motifs dans la sortie de la session et envoient une action. Limités en débit : chaque déclencheur a un temps de refroidissement de 2 s par motif, et un plafond global de 20 déclenchements par 10 s pour tous les déclencheurs d'une session.
  • Périodiques — envoient une action toutes les N secondes.
  • Actions de démarrage — envoient une action une fois à la connexion.

SMTP

Paramètres → SMTP…

  • Hôte / port
  • Sécurité — Aucune, STARTTLS ou SMTPS. Si un nom d'utilisateur est configuré, STARTTLS ou SMTPS est requis — l'authentification en texte clair est refusée pour éviter la fuite d'identifiants.
  • Nom d'utilisateur / mot de passe — le mot de passe est chiffré au repos (DPAPI sous Windows, Keychain sous macOS).
  • Adresse / nom de l'expéditeur — également utilisé comme adresse destinataire pour Envoyer à l'administrateur.
  • Délai d'expiration (s)

La validation du certificat TLS est appliquée : les certificats auto-signés, expirés ou avec un nom d'hôte non concordant sont rejetés avec un message d'erreur clair.

Journaux

  • Journaux → Ouvrir le dossier des journaux — un fichier horodaté par session.
  • Captures d'instantanés : sous-dossier snapshots/.
  • Bascule de journalisation par session dans l'onglet Général de l'éditeur de session.
  • Repli automatique : si le répertoire de journaux configuré ne peut pas être écrit (point de réanalyse OneDrive sans connexion, Controlled Folder Access ou restrictions ACL), les journaux de session sont écrits à la place dans C:\TSCRT\sessions\ sous Windows, ou ~/TSCRT/sessions/ sous macOS/Linux. Ouvrir le dossier des journaux détecte cela et ouvre automatiquement le répertoire de repli, avec un avis dans la barre d'état.

Sécurité

Vérification de la clé d'hôte SSH

TSCRT utilise un modèle Trust On First Use (TOFU). Lors de la première connexion à un serveur, la clé d'hôte est épinglée dans ~/tscrt/known_hosts (format OpenSSH) et son empreinte SHA-256 est journalisée.

  • Correspondance — la connexion se poursuit silencieusement.
  • Nouvel hôte — la clé est automatiquement épinglée et persistée.
  • Non-correspondance — la connexion est refusée avec une erreur indiquant l'empreinte du serveur et le chemin du known_hosts. Cela protège contre les attaques de type homme du milieu. Si la clé du serveur a légitimement changé, supprimez l'entrée périmée du fichier known_hosts et reconnectez-vous.

Stockage des identifiants

  • Windows — les mots de passe SSH et SMTP sont chiffrés avec DPAPI, liés au compte utilisateur actuel. Le fichier de profil est en outre protégé par une ACL réservée au propriétaire.
  • macOS — les mots de passe sont stockés dans le Keychain système (via Security.framework). Le profil ne contient qu'une référence keychain:<uuid>.
  • Linux — les mots de passe sont actuellement stockés en texte clair dans le profil (permissions 0600). Une version future pourra intégrer libsecret.
  • Les profils migrés de Windows vers macOS perdront les mots de passe chiffrés. Réintroduisez-les après la migration.

Vidages de plantage

Si TSCRT plante, un vidage est écrit dans ~/tscrt/dumps/ avec des permissions réservées au propriétaire (0600 sur POSIX, ACL réservée au propriétaire sous Windows). Les vidages de plus de 7 jours sont automatiquement supprimés au démarrage.

Support & diagnostics

Aide → Exporter les diagnostics… conditionne un fichier texte compressé en gzip (tscrt-diag-YYYYMMDD-HHMMSS.txt.gz) contenant des informations système, les chemins résolus, des variables d'environnement sélectionnées, une sonde d'écriture, les fichiers journaux récents de l'application (7 derniers jours), et les noms des éventuels vidages de plantage. La taille typique est inférieure à 1 Mo. La taille maximale avant compression est plafonnée à 10 Mo — le contenu de journal plus ancien est tronqué avec un marqueur explicite si nécessaire.

Une boîte de dialogue de consentement affiche le contenu exact avant l'écriture du bundle. Le bundle n'inclut pas les mots de passe, les clés privées SSH, le profil de session (hôtes, instantanés, règles d'automatisation), les fichiers d'instantanés, les captures de terminal par session, ni les commandes que vous avez saisies.

Examinez le fichier .txt.gz généré avant de le partager avec le support.

Importation / exportation de profil

  • Fichier → Exporter le profil… — complet ou partiel.
  • Fichier → Importer le profil… — fusionné dans le profil actuel ; les conflits sont signalés.
