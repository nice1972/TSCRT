Сеансы

Сеанс — это определение подключения: SSH или последовательное.

  • Сеансы → Управление сеансами… — добавление, редактирование, дублирование или удаление сеансов. Сеансы сгруппированы в дерево в левой панели.
  • SSH требует хост, порт, имя пользователя и либо пароль, либо файл закрытого ключа.
  • Последовательный требует имя устройства (например, COM3 в Windows, /dev/cu.usbserial-xxx в macOS), скорость в бодах, биты данных/стоп/чётности и управление потоком.
  • Дважды щёлкните по сеансу в левой панели, чтобы открыть его в новой вкладке.

Редактор сеансов (три вкладки)

Редактор сеансов разделён на три вкладки:

  • Подключение — хост/порт/имя пользователя/пароль/ключ (SSH) или устройство/скорость/настройки линии (последовательное). Эта вкладка открывается первой, чтобы наиболее часто изменяемые поля были сразу видны.
  • Общие — имя сеанса, тип подключения, тип терминала для каждого сеанса (например, xterm-256color, vt100), переключатель журналирования сеанса и параметры полноэкранного отображения. Если тип терминала оставлен пустым, используется глобальное значение по умолчанию из Настройки → Параметры.
  • Дополнительно — настройки автопереподключения (максимальное число попыток, базовая задержка) и SSH keepalive (интервал, TCP keepalive). Группа keepalive скрыта для последовательных сеансов.

Вкладки и многооконный режим

  • Каждое открытое подключение живёт в собственной вкладке. Заголовок вкладки показывает имя сеанса; маленький × рядом с именем закрывает только эту вкладку.
  • Ctrl+Alt+← / → — переход к предыдущей / следующей вкладке.
  • F11 — переключение полноэкранного режима (скрывает панель и строку меню). Все сочетания продолжают работать в полноэкранном режиме.

Контекстное меню вкладки (правый клик)

Щёлкните правой кнопкой по заголовку вкладки, чтобы получить доступ к:

  • Переименовать — изменить отображаемый заголовок вкладки.
  • Дублировать — открыть тот же сеанс в новой вкладке.
  • Запустить снимок → — запустить снимок для сеанса этой вкладки.
  • Закрепить / Открепить — закреплённые вкладки автоматически восстанавливаются при следующем запуске.
  • Отсоединить в новое окно — переместить вкладку в совершенно новое окно TSCRT. SSH/последовательный сеанс остаётся подключённым — без разрыва/переподключения. Отключено, когда только одна вкладка.
  • Закрыть — закрыть вкладку.

Перетаскивание вкладок между окнами

Перетащите вкладку из полосы вкладок и отпустите её на другом окне TSCRT, чтобы передать сеанс. Если вы отпустите её вне любого окна TSCRT, автоматически создаётся новое окно. Активное подключение сохраняется на протяжении всего процесса.

Разделение панелей / Трансляция ввода / Автопереподключение

Разделение панелей

Одна вкладка может содержать несколько терминальных панелей, расположенных в рекурсивном разделительном дереве. Разделение повторно использует профиль активной панели, так что новая панель оказывается на том же хосте.

  • Ctrl+Shift+H — разделить горизонтально (рядом).
  • Ctrl+Shift+V — разделить вертикально (сверху / снизу).
  • Ctrl+Shift+W — закрыть активную панель. Закрытие последней панели закрывает вкладку.
  • Разделения рекурсивны: горизонтальное разделение с последующим вертикальным создаёт сетку.
  • Активная панель отмечена синей рамкой 1 px. Клики мыши и фокус клавиатуры обновляют её автоматически.
  • Перетаскивайте манипулятор разделителя для изменения размера.
  • Панель поиска, Метка, Цикл и командная строка всегда действуют на активной панели.

Трансляция ввода

Зеркалируйте нажатия клавиш на несколько панелей — полезно для администрирования парка серверов.

  • Переключайте через Вид → Транслировать ввод или Ctrl+Shift+B.
  • Пока включено, каждая панель получает оранжевую рамку.
  • Нажатия клавиш, вставка, ввод IME и отчёты мыши из одной панели пересылаются во все остальные панели.
  • Переключатель для каждой вкладки.

Автопереподключение и SSH keepalive

Настраивается на вкладке Дополнительно редактора сеансов.

  • Автопереподключение — восстановить сеанс после обрыва. Вкладки, закрытые вручную, не будут переподключаться.
  • Максимум попыток — предел отказа. 0 = неограниченно (жёсткий предел 100).
  • Базовая задержка (мс) — начальный backoff (минимум 250 мс). Каждая повторная попытка удваивается, потолок 30 с, с джиттером 0–250 мс.
    Пример с базой 500: 500 мс → 1 с → 2 с → 4 с → 8 с → 16 с → 30 с …
  • SSH keepalive (с) (только SSH) — интервал keepalive libssh2. Рекомендуется 30 с.
  • TCP keepalive (только SSH) — SO_KEEPALIVE на сокете.

Во время переподключения: строка состояния показывает прогресс; буфер прокрутки сохраняется; каждая панель имеет собственное независимое состояние переподключения.

Кнопки

До 20 определяемых пользователем кнопок действий в нижней полосе. Клик для отправки, правый клик для редактирования.

Синтаксис строки действия

EscapeЗначение
\rВозврат каретки (Enter)
\nПеревод строки
\tТабуляция
\bBackspace
\eEscape (0x1B)
\pПауза 1 секунда
\\Буквальная обратная косая

Примечание: Неизвестные escape-последовательности (например, \q) молча отбрасываются и регистрируются как предупреждение. Обрабатываются только перечисленные выше escape.

Примеры

1. Отправить команду: uptime\r

2. Последовательный вход: \r\padmin\rs3cret\r (Enter → 1 с → admin+Enter → пароль+Enter)

3. Отправить Ctrl-C: \x03

4. ESC, затем диагностика: \e\pshow interfaces\r

Кнопки Цикл и Метка

  • Цикл — правый клик на пользовательской кнопке → Повторять эту кнопку для повторения с интервалом.
  • Метка — вставить маркер с отметкой времени в журнал сеанса. Левый клик = метка, правый клик = редактировать подпись.

Снимки

Снимок — это именованная последовательность команд, которая выполняется на сеансе, захватывает вывод и записывает файл с отметкой времени. При необходимости отправляет результат по электронной почте.

Настройка

Снимки → Управление снимками…

  • Имя / описание
  • Команды — каждая строка: строка команды, задержка (мс), необязательный ожидаемый prompt (регулярное выражение), максимальное ожидание (мс).
  • Получатели, шаблон темы ({session}, {snapshot}, {timestamp}).
  • Отправить письмо / Приложить файл.

Регулярное выражение ожидаемого prompt сопоставляется со скользящим окном в 64 КБ вывода шага, что предотвращает блокировку GUI патологическими шаблонами.

Выполнение

  • Меню: Снимки → <имя> или правый клик по вкладке → Запустить снимок.
  • Файлы: каталог журнала/snapshots/. Просмотр через Снимки → Обзор снимков….

Правила автоматизации

Снимки → Правила автоматизации…

  • on_connect — выполнить один раз при подключении сеанса.
  • cron — 5-полевое cron-расписание.
  • pattern — regex-совпадение в выводе сеанса, с периодом ожидания (секунды).

Триггеры, периодические действия, действия при запуске

Настраиваются в Настройки → Параметры:

  • Триггеры — определяют шаблоны в выводе сеанса и отправляют действие. Ограничены по скорости: каждый триггер имеет период ожидания 2 с на шаблон и глобальный предел 20 срабатываний за 10 с для всех триггеров в сеансе.
  • Периодические — отправляют действие каждые N секунд.
  • Действия при запуске — отправляют действие один раз при подключении.

SMTP

Настройки → SMTP…

  • Хост / порт
  • Безопасность — Нет, STARTTLS или SMTPS. Если настроено имя пользователя, требуется STARTTLS или SMTPS — аутентификация в открытом виде отклоняется во избежание утечки учётных данных.
  • Имя пользователя / пароль — пароль шифруется при хранении (DPAPI в Windows, Keychain в macOS).
  • Адрес / имя отправителя — также используется как адрес получателя для Отправить администратору.
  • Тайм-аут (с)

Применяется проверка сертификата TLS: самоподписанные, просроченные сертификаты или сертификаты с несоответствующим именем хоста отклоняются с понятным сообщением об ошибке.

Журналы

  • Журналы → Открыть папку журналов — один файл с отметкой времени на сеанс.
  • Захваты снимков: подпапка snapshots/.
  • Переключатель журналирования для каждого сеанса на вкладке Общие редактора сеансов.

Безопасность

Проверка ключа хоста SSH

TSCRT использует модель Trust On First Use (TOFU). При первом подключении к серверу ключ хоста закрепляется в ~/tscrt/known_hosts (формат OpenSSH), и его отпечаток SHA-256 записывается в журнал.

  • Совпадение — подключение продолжается безмолвно.
  • Новый хост — ключ автоматически закрепляется и сохраняется.
  • Несовпадение — подключение отклоняется с ошибкой, отображающей отпечаток сервера и путь known_hosts. Это защищает от атак "человек посередине". Если ключ сервера был законно изменён, удалите устаревшую запись из файла known_hosts и переподключитесь.

Хранение учётных данных

  • Windows — пароли SSH и SMTP шифруются с помощью DPAPI, привязанные к текущей учётной записи пользователя. Файл профиля дополнительно защищён ACL только для владельца.
  • macOS — пароли хранятся в системном Keychain (через Security.framework). Профиль содержит только ссылку keychain:<uuid>.
  • Linux — пароли в настоящее время хранятся в открытом виде в профиле (права доступа 0600). В будущей версии может быть интегрирован libsecret.
  • Профили, перенесённые с Windows на macOS, потеряют зашифрованные пароли. После миграции введите их заново.

Дампы аварийных завершений

Если TSCRT аварийно завершается, дамп записывается в ~/tscrt/dumps/ с правами доступа только для владельца (0600 на POSIX, ACL только для владельца на Windows). Дампы старше 7 дней автоматически удаляются при запуске.

Импорт / экспорт профиля

  • Файл → Экспорт профиля… — полный или частичный.
  • Файл → Импорт профиля… — сливается с текущим профилем; о конфликтах сообщается.
