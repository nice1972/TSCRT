TSCRT 사용 설명서

TSCRT는 SSH2 및 시리얼 콘솔용 터미널 에뮬레이터입니다. 주요 기능별 안내는 아래 링크에서 바로 이동할 수 있습니다.

  • 세션(Sessions)
  • 탭(Tabs)
  • 버튼(Buttons) — 예제 포함
  • 스냅샷(Snapshots) — 예제 포함
  • 자동화 규칙(Automation rules)
  • 트리거 / 주기 실행 / 시작 동작
  • SMTP (메일 발송)
  • 로그(Logs)
  • 프로필 가져오기 / 내보내기
  • 단축키

세션

세션은 SSH 또는 시리얼 접속 정의 하나를 의미합니다.

  • Sessions → Manage sessions…에서 세션을 추가·수정·복제·삭제합니다. 세션은 왼쪽 도크의 트리에 폴더별로 정리됩니다.
  • SSH: 호스트, 포트, 사용자명, 비밀번호 또는 개인키 파일이 필요합니다.
  • 시리얼: 장치명(예: Windows COM3, Linux/macOS /dev/ttyUSB0), 전송 속도, 데이터 비트, 정지 비트, 패리티, 흐름 제어가 필요합니다.
  • 왼쪽 도크의 세션을 더블 클릭하면 새 탭으로 엽니다.

탭

  • 접속 하나당 탭 하나입니다. 탭 이름 옆의 작은 ×로 해당 탭만 닫을 수 있습니다.
  • F11로 전체 화면 모드를 토글합니다 (도크와 메뉴 바 숨김).
  • 세션별로 명령 입력줄과 버튼 바 표시 여부를 세션 편집기에서 설정할 수 있습니다.

버튼

하단 버튼 바에는 최대 20개의 사용자 정의 버튼이 배치됩니다. 버튼을 누르면 설정된 액션 문자열이 현재 활성 세션으로 전송됩니다. 버튼을 우클릭하면 해당 슬롯을 편집합니다.

액션 문자열 문법

액션 문자열은 일반 텍스트와 이스케이프 시퀀스를 조합합니다.

이스케이프의미
\r캐리지 리턴 (Enter)
\n라인 피드
\t탭
\b백스페이스
\eESC (0x1B)
\p1초 대기
\\역슬래시 리터럴

예제

1. 한 줄 명령 전송 — 라벨 uptime, 액션:

uptime\r

uptime을 입력하고 Enter를 누릅니다.

2. 시리얼 콘솔 로그인 자동화 — 라벨 login, 액션:

\r\padmin\rs3cret\r

Enter → 1초 대기 → admin+Enter → s3cret+Enter 순으로 전송합니다.

3. Ctrl 키 전송 — 라벨 ^C, 액션:

\x03

\xNN 16진수 이스케이프로 임의의 바이트를 삽입할 수 있습니다.

4. 취소 후 진단 명령 실행 — 라벨 diag, 액션:

\e\pshow interfaces\r

ESC 전송 → 1초 대기 → show interfaces 실행.

Loop / Mark 특수 버튼

사용자 버튼 오른쪽에 특수 버튼 두 개가 있습니다.

  • Loop — 사용자 버튼을 우클릭 → Loop this button을 선택하면 해당 액션을 일정 간격으로 반복 실행합니다. 매 틱마다 Loop 버튼이 깜빡이며, Loop를 다시 누르면 정지합니다.
  • Mark — 세션 로그에 타임스탬프 마커 라인을 삽입합니다. 좌클릭은 즉시 마크, 우클릭은 라벨 편집.

스냅샷

스냅샷은 명령 시퀀스에 이름을 붙인 것으로, 세션에서 순서대로 실행한 출력을 캡처하여 타임스탬프가 붙은 파일로 저장합니다. 선택적으로 이메일로 전송할 수 있습니다.

스냅샷 설정

Snapshots → Manage snapshots…에서 다음 항목을 설정합니다.

  • 이름 / 설명 — Snapshots 메뉴에 표시됩니다.
  • Commands — 순서가 있는 명령 리스트. 각 행은 다음 필드를 갖습니다.
    • Command: 액션 문자열 (버튼과 동일한 문법, 예: show version\r)
    • Delay (ms): 전송 후 최소 대기 시간 — 장치가 응답할 시간을 줍니다.
    • Expect prompt: 정규식 — 매칭되면 해당 단계를 조기 종료합니다.
    • Max wait (ms): Expect prompt 사용 시 최대 대기 시간.
  • Recipients — 수신자 이메일 (한 줄에 하나).
  • Subject template — {session}, {snapshot}, {timestamp} 치환자를 사용할 수 있습니다.
  • Send email / Attach file — 메일 전송 여부 및 본문 포함 vs 파일 첨부 여부.

실행 방법

  • Snapshots → <스냅샷 이름> 메뉴로 현재 활성 세션에서 실행합니다.
  • 결과 파일은 log dir/snapshots/ 아래에 저장됩니다. Snapshots → Open snapshot folder로 탐색기를 열거나, Snapshots → Browse snapshots…로 TSCRT 내부 브라우저를 열 수 있습니다.
  • 내부 브라우저에서 파일을 우클릭하고 Send to administrator를 선택하면 SMTP의 From 주소(메일 서버에 등록된 관리자 계정)로 파일이 첨부되어 발송됩니다.

예제

1. Linux 상태 점검

name         : linux-health
description  : 부하 / 디스크 / 메모리 스냅샷
command 1    : uptime\r               delay: 500
command 2    : df -h\r                delay: 500
command 3    : free -m\r              delay: 500
command 4    : dmesg | tail -n 20\r   delay: 800
subject_tmpl : [TSCRT] {session} 상태 {timestamp}
recipients   : ops@example.com
send_email   : yes
attach_file  : yes

2. Cisco 스위치 설정 백업 — expect_prompt로 페이징 회피

name         : switch-config
command 1    : terminal length 0\r    delay: 200
command 2    : show running-config\r  delay: 0
               expect_prompt: #\s*$   max_wait: 15000
command 3    : show version\r         delay: 0
               expect_prompt: #\s*$   max_wait: 5000
subject_tmpl : config-backup {session} {timestamp}

정규식 #\s*$는 줄 끝의 스위치 프롬프트를 매칭하므로, 출력이 끝나는 즉시 다음 단계로 진행합니다.

3. 다단계 대기 포함 예제

command 1: \e             delay: 300    (ESC 전송)
command 2: menu\r         delay: 500
command 3: \p4\r          delay: 1000   (1초 대기 후 "4" + Enter)

자동화 규칙

Snapshots → Automation rules…에서 스냅샷을 트리거와 연결합니다. 세 가지 종류가 있습니다.

  • on_connect — 세션이 연결되는 순간 한 번 실행.
  • cron — 5 필드 cron (m h dom mon dow). 예: */15 * * * * = 15분마다.
  • pattern — 세션 수신 데이터가 정규식에 매칭되면 실행. 초 단위 쿨다운으로 과다 실행을 방지합니다.

규칙은 특정 세션 이름에만 적용하거나, 비워 두면 모든 세션에 적용됩니다.

트리거 / 주기 실행 / 시작 동작

Settings → Preferences에서 설정합니다.

  • Triggers — 세션 출력이 정규식에 매칭될 때 액션 문자열을 전송합니다. 프롬프트 자동 응답에 유용.
  • Periodics — 지정 세션으로 N초마다 액션을 전송합니다 (keepalive, 폴링 등).
  • Startup actions — 세션 연결 직후 한 번 액션을 전송합니다.

SMTP

Settings → SMTP…에서 스냅샷 메일 전송과 Send to administrator 기능이 사용하는 발송 서버를 설정합니다.

  • Host / Port — 예: smtp.gmail.com / 587.
  • Security — None, STARTTLS, SMTPS (암시적 TLS) 중 선택.
  • Username / Password — 비밀번호는 암호화되어 저장됩니다.
  • From address / name — From 헤더에 사용되며, 이 주소가 곧 "관리자" 주소로 Send to administrator에서 사용됩니다.
  • Timeout (s) — 소켓 단계별 타임아웃.

로그

  • Logs → Open log folder로 로그 디렉터리를 엽니다. 세션별로 타임스탬프가 붙은 파일이 생성됩니다.
  • 스냅샷 결과 파일은 snapshots/ 하위 폴더에 저장됩니다.
  • 세션별 로그 기록 여부는 세션 편집기의 Log enabled에서 켭니다.

프로필 가져오기 / 내보내기

프로필에는 세션, 버튼, 스냅샷, 규칙, SMTP 설정이 포함됩니다.

  • File → Export profile… — 전체 또는 일부 섹션만 선택하여 내보낼 수 있습니다.
  • File → Import profile… — 현재 프로필과 병합하며, 충돌이 있으면 보고합니다.

단축키

  • Ctrl+T — 현재 세션으로 새 탭 열기
  • Ctrl+W — 현재 탭 닫기
  • F11 — 전체 화면 토글
  • Ctrl+L — 명령 입력줄로 포커스
  • Ctrl+Shift+S — 스냅샷 관리 열기
