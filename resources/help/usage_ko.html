TSCRT 사용 설명서

TSCRT는 SSH2 및 시리얼 콘솔용 터미널 에뮬레이터입니다. 주요 기능별 안내는 아래 링크에서 바로 이동할 수 있습니다.

  • 세션(Sessions)
  • 탭 & 멀티 윈도우
  • 분할 창 / 입력 브로드캐스트 / 자동 재접속
  • 버튼(Buttons) — 예제 포함
  • 스냅샷(Snapshots) — 예제 포함
  • 자동화 규칙(Automation rules)
  • 트리거 / 주기 실행 / 시작 동작
  • SMTP (메일 발송)
  • 로그(Logs)
  • 보안
  • 프로필 가져오기 / 내보내기
  • 단축키

세션

세션은 SSH 또는 시리얼 접속 정의 하나를 의미합니다.

  • Sessions → Manage sessions…에서 세션을 추가·수정·복제·삭제합니다. 세션은 왼쪽 도크의 트리에 정리됩니다.
  • SSH: 호스트, 포트, 사용자명, 비밀번호 또는 개인키 파일이 필요합니다.
  • 시리얼: 장치명(예: Windows COM3, macOS /dev/cu.usbserial-xxx), 전송 속도, 데이터/정지/패리티 비트, 흐름 제어가 필요합니다.
  • 왼쪽 도크의 세션을 더블 클릭하면 새 탭으로 엽니다.

세션 편집기 (3탭 구성)

세션 편집 다이얼로그는 세 개의 탭으로 나뉩니다:

  • Connection — 호스트/포트/사용자/비밀번호/키(SSH) 또는 장치/전송속도/라인 설정(Serial). 가장 자주 변경하는 필드가 먼저 보이도록 첫 번째 탭입니다.
  • General — 세션 이름, 접속 유형, 세션별 터미널 타입(예: xterm-256color, vt100), 세션 로깅, 전체화면 표시 옵션. 터미널 타입을 비워 두면 Settings → Preferences의 글로벌 기본값이 적용됩니다.
  • Advanced — 자동 재접속(최대 횟수, 기본 지연), SSH keepalive(간격, TCP keepalive). 시리얼 세션에서는 keepalive 그룹이 숨겨집니다.

탭 & 멀티 윈도우

  • 접속 하나당 탭 하나입니다. 탭 이름 옆의 작은 ×로 해당 탭만 닫을 수 있습니다.
  • Ctrl+Alt+← / → — 이전 / 다음 탭으로 이동합니다.
  • F11 — 전체 화면 토글 (도크와 메뉴 바 숨김). 모든 단축키는 전체화면에서도 동작합니다.

탭 우클릭 메뉴

탭 제목을 우클릭하면 다음 항목을 사용할 수 있습니다:

  • Rename — 탭 표시 이름 변경.
  • Duplicate — 같은 세션을 새 탭으로 열기.
  • Run Snapshot → — 이 탭의 세션에서 스냅샷 실행.
  • Pin / Unpin — 고정된 탭은 다음 실행 시 자동으로 복원됩니다.
  • Detach to New Window — 탭을 새 TSCRT 창으로 분리합니다. SSH/시리얼 연결은 끊기지 않고 그대로 유지됩니다. 탭이 1개일 때는 비활성입니다.
  • Close — 탭 닫기.

윈도우 간 탭 드래그

탭을 드래그해서 다른 TSCRT 창의 탭 바에 드롭하면 세션이 그 창으로 이동합니다. TSCRT 창 밖으로 드롭하면 새 창이 자동 생성됩니다. 이동 중에도 연결은 끊기지 않습니다.

분할 창 / 입력 브로드캐스트 / 자동 재접속

분할 창 (Split Pane)

한 탭 안에서 터미널을 가로/세로로 분할해 여러 세션을 동시에 운용할 수 있습니다. 분할 시 같은 프로필의 세션이 새 패널에 자동으로 만들어집니다.

  • Ctrl+Shift+H — 가로(좌우) 분할
  • Ctrl+Shift+V — 세로(상하) 분할
  • Ctrl+Shift+W — 활성 패널 닫기. 마지막 패널을 닫으면 탭이 닫힙니다.
  • 분할 트리는 재귀적입니다. 가로 분할 후 한쪽을 다시 세로로 나누면 격자 레이아웃이 됩니다.
  • 활성 패널은 파란색 1px 테두리로 표시됩니다.
  • 패널 사이의 분할선은 마우스로 드래그해 비율을 조정할 수 있습니다.
  • FindBar / Mark / Loop / 명령 입력줄은 항상 활성 패널에 적용됩니다.

입력 브로드캐스트

여러 서버를 동시에 관리할 때 같은 명령을 한 번 타이핑해 모든 패널에 동시에 보내는 기능입니다.

  • View → Broadcast Input 또는 Ctrl+Shift+B로 토글합니다.
  • 활성화되면 현재 탭의 모든 패널이 주황색 테두리로 표시됩니다.
  • 한 패널에서 입력한 키, 붙여넣기, 한글 IME 조합, 마우스 리포트가 다른 모든 패널로 동시에 전송됩니다.
  • 토글은 탭 단위입니다.

자동 재접속 + Keepalive

세션 편집기의 Advanced 탭에서 설정합니다.

  • Auto reconnect — 끊겼을 때 자동 재접속. 사용자가 직접 닫은 탭은 재접속하지 않습니다.
  • Max attempts — 최대 재시도 횟수. 0 = 무제한 (하드 상한 100회).
  • Base delay (ms) — 첫 재시도 대기 (최소 250 ms). 이후 2배씩 증가, 30 s 상한, 0~250 ms 지터.
    예: base 500 → 500 ms → 1 s → 2 s → 4 s → 8 s → 16 s → 30 s …
  • SSH keepalive (s) (SSH 전용) — libssh2 keepalive 간격. 권장 30 s.
  • TCP keepalive (SSH 전용) — TCP 소켓 SO_KEEPALIVE 설정.

재접속 동작: 상태 바에 진행 표시, 스크롤백 유지, 각 패널은 독립적인 재접속 상태를 갖습니다.

버튼

하단 버튼 바에 최대 20개의 사용자 정의 버튼. 클릭하면 전송, 우클릭하면 편집합니다.

액션 문자열 문법

이스케이프의미
\r캐리지 리턴 (Enter)
\n라인 피드
\t탭
\b백스페이스
\eESC (0x1B)

참고: 위에 정의되지 않은 이스케이프 시퀀스(예: \q)는 무시되고 경고가 기록됩니다.

예제

1. 명령 전송: uptime\r

2. 시리얼 로그인: \r\padmin\rs3cret\r (Enter → 1초 → admin+Enter → 비밀번호+Enter)

3. Ctrl-C 전송: \x03

4. ESC 후 진단: \e\pshow interfaces\r

Loop / Mark 특수 버튼

  • Loop — 사용자 버튼 우클릭 → Loop this button으로 일정 간격 반복 실행.
  • Mark — 세션 로그에 타임스탬프 마커 삽입. 좌클릭 = 즉시 마크, 우클릭 = 라벨 편집.

스냅샷

스냅샷은 명령 시퀀스에 이름을 붙인 것으로, 세션에서 순서대로 실행한 출력을 캡처하여 파일로 저장합니다.

스냅샷 설정

Snapshots → Manage snapshots…

  • 이름 / 설명
  • Commands — 각 행: 커맨드, 지연(ms), expect prompt(정규식), max wait(ms).
  • Recipients, Subject template ({session}, {snapshot}, {timestamp}).
  • Send email / Attach file.

expect_prompt 정규식은 해당 단계 출력의 최근 64 KB 롤링 윈도우에서 매칭됩니다.

실행 방법

  • 메뉴: Snapshots → <스냅샷 이름>, 또는 탭 우클릭 → Run Snapshot.
  • 파일: log dir/snapshots/. Snapshots → Browse snapshots…로 탐색.

자동화 규칙

Snapshots → Automation rules…

  • on_connect — 세션 연결 시 한 번 실행.
  • cron — 5필드 cron 스케줄.
  • pattern — 세션 수신 데이터 정규식 매칭 시 실행. 초 단위 쿨다운.

트리거 / 주기 실행 / 시작 동작

Settings → Preferences에서 설정합니다.

  • Triggers — 세션 출력 패턴 매칭 시 액션 전송. 보안을 위해 rate-limit 적용: 패턴당 2초 쿨다운, 전체 세션당 10초에 최대 20회.
  • Periodics — 지정 세션으로 N초마다 액션 전송.
  • Startup actions — 세션 연결 직후 한 번 액션 전송.

SMTP

Settings → SMTP…

  • Host / Port
  • Security — None, STARTTLS, SMTPS. 사용자명이 설정되어 있으면 STARTTLS 또는 SMTPS 필수 — 평문 인증은 거부됩니다.
  • Username / Password — 비밀번호는 암호화 저장 (Windows: DPAPI, macOS: Keychain).
  • From address / name — Send to administrator 대상 주소로도 사용됩니다.
  • Timeout (s)

TLS 인증서 검증이 강제됩니다: 자가서명, 만료, 호스트명 불일치 인증서는 명확한 에러와 함께 거부됩니다.

로그

  • Logs → Open log folder — 세션별 타임스탬프 파일.
  • 스냅샷 결과: snapshots/ 하위 폴더.
  • 세션별 로그 기록 여부는 세션 편집기 General 탭에서 설정.

보안

SSH 호스트 키 검증

TSCRT는 TOFU(Trust On First Use) 모델을 사용합니다. 서버에 처음 접속하면 호스트 키가 ~/tscrt/known_hosts(OpenSSH 형식)에 고정되고 SHA-256 지문이 기록됩니다.

  • 일치 — 조용히 통과.
  • 신규 호스트 — 자동으로 키 고정 및 저장.
  • 불일치 — 연결 거부. 서버 지문과 known_hosts 경로가 에러에 표시됩니다. 중간자 공격 방어. 서버 키가 정당하게 변경되었다면 known_hosts에서 해당 항목을 제거하고 재접속하세요.

자격 증명 저장

  • Windows — DPAPI 암호화, 현재 사용자 계정에 바인딩. 프로파일 파일에 owner-only ACL 적용.
  • macOS — 시스템 키체인(Security.framework)에 저장. 프로파일에는 keychain:<uuid> 참조만 기록.
  • Linux — 현재 평문 저장 (0600 권한). 향후 libsecret 통합 예정.
  • Windows에서 macOS로 프로파일을 이동하면 암호화된 비밀번호가 풀리지 않습니다. 이동 후 재입력하세요.

크래시 덤프

충돌 시 ~/tscrt/dumps/에 덤프가 기록됩니다 (POSIX: 0600, Windows: owner-only ACL). 7일 이상 된 덤프는 시작 시 자동 삭제됩니다.

프로필 가져오기 / 내보내기

  • File → Export profile… — 전체 또는 부분.
  • File → Import profile… — 현재 프로필과 병합, 충돌 보고.
