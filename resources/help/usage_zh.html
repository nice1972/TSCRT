TSCRT 使用指南

TSCRT 是一款面向 SSH2 与串口控制台的终端仿真器。本指南介绍所有主要功能。

  • 会话(Sessions)
  • 标签页(Tabs)
  • 按钮(Buttons)（含示例）
  • 快照(Snapshots)（含示例）
  • 自动化规则
  • 触发器 / 周期任务 / 启动动作
  • SMTP（邮件发送）
  • 日志
  • 配置文件导入 / 导出
  • 快捷键

会话

会话是一条 SSH 或串口连接的定义。

  • Sessions → Manage sessions… 可添加、编辑、复制、删除会话。会话按分组在左侧停靠栏的树中显示。
  • SSH 需要主机、端口、用户名，以及密码或私钥文件。
  • 串口 需要设备名（Windows 如 COM3，Linux/macOS 如 /dev/ttyUSB0）、波特率、数据位、停止位、校验位和流控。
  • 双击左侧停靠栏中的会话以在新标签页中打开。

标签页

  • 每个连接占一个标签页。标签名旁边的小 × 只关闭该标签。
  • F11 切换全屏模式（隐藏停靠栏和菜单栏）。
  • 每个会话可以在会话编辑器中设置是否显示命令行和按钮栏。

按钮

底部按钮栏最多可放置 20 个用户自定义按钮。点击按钮会将其动作字符串发送到当前活动会话；右键点击按钮可编辑该槽位。

动作字符串语法

转义含义
\r回车 (Enter)
\n换行
\t制表符
\b退格
\eESC (0x1B)
\p暂停 1 秒
\\反斜杠字面量

示例

1. 发送一条命令 — 标签 uptime，动作:

uptime\r

输入 uptime 然后按回车。

2. 串口控制台登录自动化 — 标签 login，动作:

\r\padmin\rs3cret\r

依次发送：回车 → 等待 1 秒 → admin+回车 → s3cret+回车。

3. 发送 Ctrl 键 — 标签 ^C，动作:

\x03

可用 \xNN 十六进制转义嵌入任意字节。

4. 取消后执行诊断 — 标签 diag，动作:

\e\pshow interfaces\r

发送 ESC → 等待 1 秒 → 执行 show interfaces。

Loop / Mark 特殊按钮

  • Loop — 右键点击任一用户按钮并选择 Loop this button，即可按固定间隔重复其动作。每次触发时 Loop 按钮会闪烁；再次点击 Loop 即可停止。
  • Mark — 在会话日志中插入带时间戳的标记行。左键立即标记，右键编辑标签文字。

快照

快照是一组有序命令的集合，TSCRT 会在会话上依次执行它们、捕获输出，并保存为带时间戳的文件；并可选地通过邮件发送结果。

配置快照

打开 Snapshots → Manage snapshots…，为每个快照配置：

  • 名称 / 描述 — 显示在 Snapshots 菜单中。
  • Commands — 有序命令列表。每行包括：
    • Command：动作字符串（与按钮同样的语法，例如 show version\r）
    • Delay (ms)：发送后的最小等待时间，留出设备响应时间
    • Expect prompt：可选正则表达式，匹配后提前结束该步骤
    • Max wait (ms)：设置了 Expect prompt 时的最长等待时间
  • Recipients — 每行一个收件人邮箱。
  • Subject template — 可使用 {session}、{snapshot}、{timestamp} 占位符。
  • Send email / Attach file — 是否发送邮件，以及以附件发送还是直接放入正文。

运行方式

  • 菜单 Snapshots → <快照名> 在当前活动会话上运行。
  • 输出文件保存到 log dir/snapshots/ 下。可通过 Snapshots → Open snapshot folder 用资源管理器打开，或通过 Snapshots → Browse snapshots… 在 TSCRT 内置浏览器中查看。
  • 在内置浏览器中右键点击文件选择 Send to administrator，即可将该文件作为附件发送到 SMTP 的 From 地址（邮件服务器上注册的管理员账户）。

示例

1. Linux 健康检查

name         : linux-health
description  : 负载 / 磁盘 / 内存 快照
command 1    : uptime\r               delay: 500
command 2    : df -h\r                delay: 500
command 3    : free -m\r              delay: 500
command 4    : dmesg | tail -n 20\r   delay: 800
subject_tmpl : [TSCRT] {session} health {timestamp}
recipients   : ops@example.com
send_email   : yes
attach_file  : yes

2. Cisco 交换机配置备份 — 用 expect_prompt 规避分页

name         : switch-config
command 1    : terminal length 0\r    delay: 200
command 2    : show running-config\r  delay: 0
               expect_prompt: #\s*$   max_wait: 15000
command 3    : show version\r         delay: 0
               expect_prompt: #\s*$   max_wait: 5000
subject_tmpl : config-backup {session} {timestamp}

正则 #\s*$ 匹配行尾的交换机提示符，输出完成即刻进入下一步。

3. 含多段等待的示例

command 1: \e             delay: 300    (发送 ESC)
command 2: menu\r         delay: 500
command 3: \p4\r          delay: 1000   (等待 1 秒后发送 "4"+回车)

自动化规则

Snapshots → Automation rules… 将快照绑定到触发条件。有三种：

  • on_connect — 会话连接时立即执行一次。
  • cron — 标准五字段 cron（m h dom mon dow）。例：*/15 * * * * 每 15 分钟执行一次。
  • pattern — 当会话的接收数据与正则匹配时执行。可通过秒级冷却时间防止过度触发。

规则可以限定到特定会话名，也可以留空以应用到所有会话。

触发器 / 周期任务 / 启动动作

在 Settings → Preferences 中配置：

  • Triggers — 当会话输出匹配某正则时发送动作字符串。适合自动应答提示。
  • Periodics — 每 N 秒向指定会话发送一次动作（心跳、轮询等）。
  • Startup actions — 会话连接后立即发送一次动作。

SMTP

Settings → SMTP… 配置快照发送邮件和 Send to administrator 所使用的发送服务器。

  • Host / Port — 如 smtp.gmail.com / 587。
  • Security — 可选 None、STARTTLS、SMTPS（隐式 TLS）。
  • Username / Password — 密码加密存储。
  • From address / name — 出现在 From 头中；这个地址同时作为"管理员"地址，供 Send to administrator 使用。
  • Timeout (s) — 每步套接字超时。

日志

  • Logs → Open log folder 打开日志目录，每个会话生成一份带时间戳的日志文件。
  • 快照结果文件保存在 snapshots/ 子目录。
  • 是否按会话记录日志在会话编辑器中通过 Log enabled 开启。

配置文件导入 / 导出

配置文件包含会话、按钮、快照、规则以及 SMTP 设置。

  • File → Export profile… — 可选择全部或部分区段导出。
  • File → Import profile… — 合并到当前配置文件，冲突会被报告。

快捷键

  • Ctrl+T — 以当前会话新建标签页
  • Ctrl+W — 关闭当前标签页
  • F11 — 切换全屏
  • Ctrl+L — 聚焦命令行
  • Ctrl+Shift+S — 打开快照管理
