会话

会话是一个连接定义,可以是 SSH 或串行。

  • 会话 → 管理会话… — 添加、编辑、复制或删除会话。会话在左侧停靠面板中以树形分组。
  • SSH 需要主机、端口、用户名以及密码或私钥文件。
  • 串行 需要设备名称(例如 Windows 上的 COM3,macOS 上的 /dev/cu.usbserial-xxx)、波特率、数据/停止/奇偶校验位和流控制。
  • 双击左侧停靠面板中的会话,可在新标签页中打开。

会话编辑器(三个选项卡)

会话编辑器分为三个选项卡:

  • 连接 — 主机/端口/用户名/密码/密钥(SSH)或设备/波特率/线路设置(串行)。此选项卡会首先打开,以便最常更改的字段立即可见。
  • 常规 — 会话名称、连接类型、每会话的终端类型(例如 xterm-256color、vt100)、会话日志开关,以及全屏显示选项。如果终端类型留空,将使用 设置 → 首选项 中的全局默认值。
  • 高级 — 自动重连设置(最大尝试次数、基础延迟)和 SSH keepalive(间隔、TCP keepalive)。串行会话将隐藏 keepalive 组。

标签页与多窗口

  • 每个打开的连接都位于自己的标签页中。标签页标题显示会话名称;名称旁的小 × 仅关闭该标签页。
  • Ctrl+Alt+← / → — 切换到上一个 / 下一个标签页。
  • F11 — 切换全屏(隐藏停靠面板和菜单栏)。全屏下所有快捷键仍然有效。

标签页上下文菜单(右键)

右键点击标签页标题以访问:

  • 重命名 — 更改显示的标签页标题。
  • 复制 — 在新标签页中打开相同的会话。
  • 运行快照 → — 在此标签页的会话上运行快照。
  • 固定 / 取消固定 — 固定的标签页会在下次启动时自动恢复。
  • 分离到新窗口 — 将标签页移动到一个全新的 TSCRT 窗口。SSH/串行会话保持连接 — 无断开/重连。仅有一个标签页时禁用。
  • 将标签页链接到对等标签页… — 将此标签页与同一台机器上的另一个 TSCRT 实例中的标签页配对。激活任一标签页也会切换另一个窗口中的对等标签页。适用于并排布局(例如,左侧显示器显示控制台,右侧显示日志查看器)。
  • 取消链接标签页 — 当标签页已经有对等标签时,替代链接显示;解除配对。
  • 关闭 — 关闭标签页。

在窗口之间拖动标签页

从标签栏拖动一个标签页,并将其放到另一个 TSCRT 窗口上以传输会话。如果放在任何 TSCRT 窗口外,会自动创建新窗口。整个过程中活动连接保持不变。

标签页链接

标签页链接在同一台机器上运行的两个 TSCRT 窗口之间同步标签页的激活。当您在一个窗口中切换到已链接的标签页时,另一个窗口中配对的标签页会自动切换到前台。此功能专为双显示器工作流设计 — 例如,左侧屏幕显示设备控制台,右侧屏幕显示日志查看器或第二台设备。

创建链接

  1. 打开两个 TSCRT 实例(第一个成为角色 A,第二个自动成为角色 B)。
  2. 在每个窗口中打开所需的会话标签页。
  3. 右键单击任一窗口中的标签页 → 将标签页链接到对等标签页…
  4. 在对话框中,列出了另一个 TSCRT 实例及其打开的标签页。同名会话的标签页通过槽位编号(0、1、…)加以区分。选择要配对的确切对等标签页,然后单击确定。
  5. 两个已链接的标签页上都会出现一个小的链条图标,确认配对成功。

您可以创建多个链接 — 例如,链接标签页 1↔3 和标签页 2↔4,形成两对相关会话。

激活同步的工作原理

  • 每个链接由唯一的配对 ID 标识。当您激活一个已链接的标签页时,TSCRT 通过本地 IPC 通道(Unix 套接字 / Windows 命名管道)向每个已连接的对等方广播配对 ID。
  • 接收方的对等进程在其绑定表中查找配对 ID,找到匹配的本地标签页并自动切换到该标签页。对等窗口也会被提升到前台。
  • 防循环保护确保激活不会回弹:当标签页因收到的消息而被切换时,出站广播会被暂时抑制。
  • 链接是角色感知的 — 每个端点记录其属于哪个进程角色(A 或 B),因此跨实例的重复会话名称永远不会导致歧义绑定。

保存与恢复

  • 链接存储在配置文件中,在重启后仍然保留。每个链接记录其配对 ID、两个端点的会话名称、槽位索引和进程角色。
  • 每个进程还将其打开的标签页列表保存到配置文件目录中的布局文件(layout_A.lst / layout_B.lst)。
  • 下次启动时,角色 A 自动重新打开其保存的标签页。如果存在链接且 layout_B.lst 非空,则会自动生成第二个 TSCRT 进程(--role=B),重新打开 B 侧标签页并重新建立所有链接绑定 — 整个双窗口环境只需一次点击即可恢复。

取消链接

右键单击已链接的标签页 → 取消链接标签页(当前:…)。链条图标消失,配对从配置文件中移除。如果所有链接都被移除,下次启动将不再自动生成第二个 TSCRT。

对等发现

每个 TSCRT 进程注册在 $XDG_RUNTIME_DIR/tscrt/peers/(Linux)或等效的临时目录(Windows / macOS)下。一个轻量级的发现定时器每 3 秒检查一次新的对等方。已崩溃进程的过时条目在套接字不可达时会被自动清理。没有任何网络流量离开本地机器。

分割窗格 / 输入广播 / 自动重连

分割窗格

单个标签页可以容纳以递归分割器树形式排列的多个终端窗格。分割重用活动窗格的配置文件,因此新窗格位于同一主机上。

  • Ctrl+Shift+H — 水平分割(并排)。
  • Ctrl+Shift+V — 垂直分割(上下)。
  • Ctrl+Shift+W — 关闭活动窗格。关闭最后一个窗格将关闭标签页。
  • 分割是递归的:水平分割后再垂直分割可生成网格。
  • 活动窗格用 1 px 蓝色边框 标识。鼠标点击和键盘焦点会自动更新它。
  • 拖动分割器手柄以调整大小。
  • 查找栏、标记、循环和命令行始终作用于活动窗格。

输入广播

将按键镜像到多个窗格 — 在管理服务器群时非常有用。

  • 通过 视图 → 广播输入 或 Ctrl+Shift+B 切换。
  • 启用时,每个窗格都会显示 橙色边框。
  • 从一个窗格来的按键、粘贴、输入法输入和鼠标报告会转发到所有其他窗格。
  • 该开关是按标签页设置的。

自动重连与 SSH keepalive

在会话编辑器的高级选项卡中配置。

  • 自动重连 — 在断开后恢复会话。手动关闭的标签页不会重连。
  • 最大尝试次数 — 放弃阈值。0 = 无限(硬性上限 100)。
  • 基础延迟(毫秒) — 初始退避(最小 250 毫秒)。每次重试加倍,上限 30 秒,带 0–250 毫秒抖动。
    基数 500 的示例: 500 毫秒 → 1 秒 → 2 秒 → 4 秒 → 8 秒 → 16 秒 → 30 秒 …
  • SSH keepalive(秒)(仅 SSH) — libssh2 keepalive 间隔。建议 30 秒。
  • TCP keepalive(仅 SSH) — 套接字上的 SO_KEEPALIVE。

重连期间: 状态栏显示进度;滚动缓冲保留;每个窗格都有自己独立的重连状态。

按钮

底部栏中最多 20 个用户定义的操作按钮。点击发送,右键编辑。

操作字符串语法

转义含义
\r回车(Enter)
\n换行
\t制表符
\b退格
\eEscape(0x1B)
\p暂停 1 秒
\\字面反斜杠

注意: 未知的转义序列(例如 \q)会被静默丢弃并记录为警告。只处理上面列出的转义。

示例

1. 发送命令: uptime\r

2. 串行登录: \r\padmin\rs3cret\r(Enter → 1 秒 → admin+Enter → 密码+Enter)

3. 发送 Ctrl-C: \x03

4. 先 ESC 后诊断: \e\pshow interfaces\r

循环按钮和标记按钮

  • 循环 — 右键单击用户按钮 → 循环此按钮以按间隔重复。
  • 标记 — 在会话日志中插入带时间戳的标记。左键单击 = 标记,右键单击 = 编辑标签。

快照

快照是一个已命名的命令序列,在会话上运行,捕获输出并写入带时间戳的文件。配置后,TSCRT 会通过内置 SMTP 客户端自动将捕获的结果通过电子邮件发送。

配置

快照 → 管理快照…

  • 名称 / 描述
  • 命令 — 每行:命令字符串、延迟(毫秒)、可选的期望提示(正则表达式)、最大等待时间(毫秒)。
  • 收件人 — 一个或多个自动接收快照结果的电子邮件地址。
  • 主题模板 — 支持占位符:{session}、{snapshot}、{timestamp}。
  • 发送电子邮件 — 按快照切换自动邮件发送的开/关。
  • 附加文件 — 启用后,捕获的输出文件将附加到电子邮件中。

期望提示的正则表达式会与步骤输出的滚动 64 KB 窗口进行匹配,这可以防止病态模式阻塞 GUI。

自动邮件发送

当快照启用了发送电子邮件且已配置 SMTP(参见 SMTP)时,TSCRT 在执行完成后立即自动将快照结果发送给所有列出的收件人。与自动化规则(on_connect / cron / pattern)结合使用,可实现完全无人值守的监控:TSCRT 按计划或触发器运行快照、捕获输出并通过电子邮件发送给团队 — 无需手动干预。

运行

  • 菜单: 快照 → <名称>,或右键单击标签页 → 运行快照。
  • 文件: 日志目录/snapshots/。通过 快照 → 浏览快照… 浏览。

自动化规则

快照 → 自动化规则…

  • on_connect — 会话连接时运行一次。
  • cron — 5 字段 cron 计划。
  • pattern — 对会话输出进行正则表达式匹配,带冷却时间(秒)。

触发器、周期性动作、启动动作

在 设置 → 首选项 中配置:

  • 触发器 — 匹配会话输出中的模式并发送动作。速率受限:每个触发器的每个模式有 2 秒冷却时间,会话中所有触发器每 10 秒共有 20 次触发的全局上限。
  • 周期性 — 每 N 秒发送一次动作。
  • 启动动作 — 连接时发送一次动作。

SMTP

设置 → SMTP…

  • 主机 / 端口
  • 安全性 — 无、STARTTLS 或 SMTPS。如果配置了用户名,则需要 STARTTLS 或 SMTPS — 拒绝明文身份验证以防止凭据泄漏。
  • 用户名 / 密码 — 密码静态加密(Windows 上使用 DPAPI,macOS 上使用 Keychain)。
  • 发件人地址 / 名称 — 也用作发送给管理员的目标地址。
  • 超时(秒)

强制执行 TLS 证书验证:自签名、已过期或主机名不匹配的证书将被拒绝,并显示清晰的错误消息。

日志

  • 日志 → 打开日志文件夹 — 每个会话一个带时间戳的文件。
  • 快照捕获: snapshots/ 子文件夹。
  • 会话编辑器的常规选项卡中有每会话日志开关。
  • 自动回退: 如果配置的日志目录无法写入(未登录的 OneDrive reparse point、Controlled Folder Access 或 ACL 限制),会话日志将改写到 Windows 上的 C:\TSCRT\sessions\,或 macOS/Linux 上的 ~/TSCRT/sessions/。打开日志文件夹 会检测到这种情况并自动打开回退目录,并在状态栏显示提示。

安全

SSH 主机密钥验证

TSCRT 使用 Trust On First Use (TOFU) 模型。首次连接到服务器时,主机密钥将固定到 ~/tscrt/known_hosts(OpenSSH 格式),其 SHA-256 指纹将被记录。

  • 匹配 — 连接静默继续。
  • 新主机 — 密钥自动固定并持久化。
  • 不匹配 — 连接被拒绝,并显示错误,指出服务器指纹和 known_hosts 路径。这可以防止中间人攻击。如果服务器密钥确实发生了合法变更,请从 known_hosts 文件中删除过时条目并重新连接。

凭据存储

  • Windows — SSH 和 SMTP 密码使用与当前用户帐户绑定的 DPAPI 加密。配置文件另外由仅限所有者的 ACL 保护。
  • macOS — 密码存储在系统 Keychain 中(通过 Security.framework)。配置文件仅包含 keychain:<uuid> 引用。
  • Linux — 密码目前以明文形式存储在配置文件中(0600 权限)。未来版本可能会集成 libsecret。
  • 从 Windows 迁移到 macOS 的配置文件将丢失加密的密码。迁移后请重新输入。

崩溃转储

如果 TSCRT 崩溃,转储将以仅限所有者的权限(POSIX 上为 0600,Windows 上为仅限所有者的 ACL)写入 ~/tscrt/dumps/。超过 7 天的转储会在启动时自动删除。

支持 & 诊断

帮助 → 导出诊断… 会打包一个 gzip 压缩的文本文件(tscrt-diag-YYYYMMDD-HHMMSS.txt.gz),其中包含系统信息、已解析的路径、所选环境变量、可写性探测、最近的应用日志文件(最近 7 天)以及任何崩溃转储的名称。典型大小小于 1 MB。压缩前最大大小上限为 10 MB — 如果需要,较旧的日志内容会带有明确标记地被截断。

在写入包之前,同意对话框会显示确切的内容。包中不包括密码、SSH 私钥、会话配置文件(主机、快照、自动化规则)、快照文件、每会话的终端捕获,或您输入的命令。

在与支持人员共享生成的 .txt.gz 之前请检查其内容。

配置文件导入 / 导出

  • 文件 → 导出配置文件… — 完整或部分。
  • 文件 → 导入配置文件… — 合并到当前配置文件;报告冲突。

更新检查

TSCRT 在启动时通过查询 GitHub Releases API 自动检查新版本。不会发送任何个人数据 — 仅向公共发布端点发出一次 HTTP GET 请求。

工作原理

  1. 启动时,TSCRT 向 api.github.com/repos/nice1972/TSCRT/releases/latest 发送一次后台请求。
  2. 响应中包含最新发布标签(例如 v1.0.10)。
  3. TSCRT 将其与编译时内置的版本(TSCRT_VERSION)进行比较。
  4. 如果远程版本更新,将出现对话框:
    TSCRT 有新版本可用:1.0.10
    (当前:1.0.8)

    是否打开下载页面?
  5. 点击是会在默认浏览器中打开 GitHub Releases 页面,您可以在其中下载适用于您平台的安装程序。
  6. 点击否关闭对话框;TSCRT 正常继续运行。

升级方法

平台升级步骤
Ubuntu / Debiansudo apt install ./tscrt_<version>_amd64.deb — 覆盖之前的版本。
Windows运行新的 .exe 安装程序;它会自动替换现有安装。
macOS打开新的 .dmg,将 tscrt_mac.app 拖到 /Applications,替换旧版本。

您的配置文件、会话日志、标签页链接以及存储在 ~/.config/tscrt/(Linux)、%APPDATA%\tscrt\(Windows)或 ~/Library/Application Support/tscrt/(macOS)下的所有用户数据在升级时都会保留 — 仅替换二进制文件。

离线 / 受限环境

如果网络请求失败(离线、DNS 阻止、代理、防火墙),检查将被静默跳过。TSCRT 绝不会在等待响应时阻塞启动。
